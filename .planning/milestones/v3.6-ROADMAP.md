**Status:** SHIPPED 2026-06-20

## Overview

Administrators now have MongoDB-backed operational visibility into QSO ingestion, bridge connectivity, authentication/admin activity, and service health without exposing secrets or changing existing QSO behavior.

## Phases

**Goal:** Add MongoDB-backed log models, reusable logger service, level thresholding, masking, retention, indexes, and live broadcast plumbing.
**Depends on:** Phase 61/62 per-user QSO collection routing, Phase 63 ACLog full-record import, Phase 64 manual ACLog sync, Phase 66 ACLog operator identity routing
**Plans:** 1 plan

Plans:

- [x] 67-01-PLAN.md - Internal log models/service, admin viewer/API, instrumentation, docs, and tests.

5. Focused tests cover thresholding, masking, retention fields, and indexes.

## Milestone Summary

**Key Decisions:**

- Use MongoDB TTL expiry via `expires_at` so application logs cannot grow forever.
- Keep internal logging failure-isolated so observability cannot break QSO ingestion, bridge, or admin workflows.
- Reuse existing admin auth and UI patterns for the Logs page rather than introducing a new admin surface.
- Preserve source Markdown as the committed documentation source and restore generated MkDocs `site/` churn after strict verification.
- Use near-live polling as a fallback for cross-process logs while preserving immediate SSE insertion where available.
- Treat contacted-station `CALL` only as QSO context; local station identity remains `MYCALL`/setup Call/station fields and operator identity remains `OPERATOR`.

**Key Accomplishments:**

- Added `app/internal_logs/` with Beanie models, thresholding, masking, retention, query helpers, and live broadcast support.
- Added admin Logs settings/viewer with filters, Previous/Next pagination, formatted collapsed JSON details, and live/near-live updates.
- Instrumented service startup/shutdown, MongoDB lifecycle, QSO writes/imports, UDP callbacks, ACLog bridge/manual sync, auth/token/admin actions, backup/restore, and log settings.
- Added focused tests for logger thresholding/masking/retention, admin log API/UI behavior, import/manual-sync/auth/token logging, UDP callback logging, and regression safety.
- Updated admin documentation with log levels, retention, filtering, event coverage, masking behavior, and `CALL`/`MYCALL`/`OPERATOR` semantics.
- Completed UAT, security review, Nyquist validation, and milestone audit with no critical gaps.
