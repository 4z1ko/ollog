## E2E Flow Verification

### Flow 1: Admin creates operator → operator logs QSO → QSO appears in log view

Status: **COMPLETE**

1. Admin logs in at `/admin/ui/login` — `POST /admin/ui/login` verifies password via `verify_password`, checks `role == "admin"`, sets HttpOnly JWT cookie, redirects to `/admin/ui/users`
2. Admin creates operator — `POST /admin/ui/users/create` calls `User.insert()`, returns `users_table.html` partial
3. Operator logs in at `/log/login` — `POST /log/login` checks `user.enabled`, verifies password, sets HttpOnly JWT cookie, redirects to `/log/`
4. Operator submits QSO form — `POST /log/qsos` calls `build_qso_dict()`, `find_duplicate()`, `QSO.insert()`, returns `log/qso_result.html` partial with success message
5. Operator navigates to `/log/view` — `GET /log/view` calls `get_qso_page()` with `_operator` filter, renders `log/log.html` with QSO in table
6. Enable/disable check: `get_current_user_cookie()` in `app/auth/dependencies.py` line 87 checks `user.enabled` on every request — a disabled operator is rejected at the dependency level

### Flow 2: Operator imports ADIF file → import report shows counts

Status: **COMPLETE** (functionality); **BROKEN** (discoverability — no nav link)

1. Operator must navigate directly to `/log/import` — no nav link exists
2. `GET /log/import` renders `templates/log/import.html` — form uses `hx-post="/log/import"` with `enctype="multipart/form-data"` and `hx-encoding="multipart/form-data"`
3. `POST /log/import` reads file bytes, calls `process_import(raw, callsign)` in `app/adif/router.py`
4. `process_import` calls `parse_adi()`, validates required fields, calls `build_qso_dict()`, calls `find_duplicate()`, calls `QSO.insert()` for accepted records
5. Returns report dict; `templates/log/import_report.html` renders accepted/duplicate/error tables with counts
6. Full data path intact; only the navigation entry point is missing

### Flow 3: Operator exports ADIF → file downloads

Status: **COMPLETE** (functionality); **BROKEN** (discoverability — no nav link)

1. Operator must navigate directly to `/log/export`
2. `GET /log/export` in `app/qso/ui_router.py` (line 484): queries `QSO.find({"_operator": callsign, "_deleted": False})`, builds `StreamingResponse` using `_qso_to_adif_dict()` + `serialize_adi()` per record
3. `Content-Disposition: attachment; filename={callsign}_logbook.adi` triggers browser download
4. ADIF header `<ADIF_VER:5>3.1.4\n<PROGRAMID:5>ollog\n<EOH>` prepended before records

### Flow 4: Two operators log simultaneously → both QSOs attributed correctly

Status: **COMPLETE**

1. Both operators authenticated independently via separate cookies with distinct JWT payloads (`sub`, `callsign`, `role`)
2. `get_current_operator_callsign_cookie` resolves to each operator's own callsign from their own JWT — never from request body
3. `build_qso_dict()` injects `operator_callsign` (serialized as `_operator`) from the JWT-resolved callsign
4. MongoDB `qsos` collection has `operator_idx` (`_operator` ASC) and `operator_active_idx` (`_operator`, `_deleted` ASC) compound indexes
5. All queries filter by `{"_operator": operator, ...}` — cross-operator data leakage is structurally prevented
6. `find_duplicate()` also scopes by `_operator` — duplicate detection is per-operator, not global

### Flow 5: Operator A logs QSO → appears in Operator B's station feed

Status: **COMPLETE** (implementation correct; requires replica set at runtime)

1. `app/main.py` lifespan starts `watch_qsos(collection, feed_manager, _templates)` as asyncio task
2. `watch_qsos` in `app/feed/manager.py` opens change stream on `qsos` collection filtered to `operationType: insert`
3. On insert, renders `templates/log/feed_row.html` with `call`, `band`, `mode`, `freq`, `operator`, `qso_date_utc` from `change["fullDocument"]`
4. `feed_manager.broadcast(html)` puts rendered HTML into every connected `asyncio.Queue`
5. `GET /feed/station` in `app/feed/router.py` yields `ServerSentEvent(data=html, event="new_qso")` for each queue item
6. `templates/log/form.html` (line 95–108): `<div hx-ext="sse" sse-connect="/feed/station">` with `<tbody id="station-feed" sse-swap="new_qso" hx-swap="afterbegin">` — new rows prepended in real time
7. Feed shows all operators' QSOs (no callsign filter on change stream or SSE endpoint) — Operator B sees Operator A's QSOs
8. Runtime dependency: MongoDB must be running as replica set `rs0` for change streams. `docker-compose.yml` configures `--replSet rs0` with `rs.initiate()` health check; `MONGODB_URI=mongodb://mongodb:27017/?replicaSet=rs0` passed to API container

## Admin API Route Name Note

The phase spec document (provided to this audit) describes the admin API as `POST /admin/api/operators` and `PATCH /admin/api/operators/{id}`. The actual implementation uses `POST /admin/users/` and `PATCH /admin/users/{username}/enabled`. This is a spec/documentation discrepancy, not a code bug — the SUMMARY for phase 02-01 correctly documents the actual path as `/admin/users`. The admin UI templates call `/admin/ui/users/create` and `/admin/ui/users/{username}/toggle` (the HTMX UI paths), which are correct. No integration break results from this naming difference.

## Infrastructure Wiring

### Docker Compose → MongoDB Replica Set

- `docker-compose.yml` line 8: `command: ["--replSet", "rs0", "--bind_ip_all", "--port", "27017"]`
- Health check runs `rs.initiate(...)` if `rs.status()` throws — ensures replica set is initialized before API starts
- `api` service `depends_on: mongodb: condition: service_healthy` — API waits for replica set to be ready
- `MONGODB_URI=mongodb://mongodb:27017/?replicaSet=rs0` — correct replica set URI passed to application
- `app/database.py`: `AsyncMongoClient(settings.mongodb_uri)` — uses the URI with `replicaSet=rs0`
- Change streams require replica set — wiring is complete

### Lifespan Startup Sequence

`app/main.py` lifespan (lines 48–68):
1. `init_db()` — connects `AsyncMongoClient`, calls `init_beanie([QSO, User])`
2. `_bootstrap_admin()` — creates initial admin from env vars if not present (idempotent)
3. `get_client()` → `watch_qsos(collection, feed_manager, _templates)` — starts change stream watcher task
4. On shutdown: cancels watcher task, calls `close_db()`

The `get_client()` guard (line 55: `if client is not None`) means the change stream watcher is skipped gracefully if MongoDB is unavailable — prevents startup crash in offline dev.

### Beanie Document Registration

`app/database.py` registers exactly `[QSO, User]` — both models used across all phases. No Beanie Document exists that is not registered.
