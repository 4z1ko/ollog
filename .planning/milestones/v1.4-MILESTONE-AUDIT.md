A single UDP datagram cannot carry more than ~64 KB at the OS level, so no UDP-specific guard is
needed.

### Missing Connections

### Broken Flows

None. The full UDP datagram → MongoDB insert → SSE broadcast pipeline is wired end-to-end. The

1. **Live socket bind** — requires a running process. Code path is fully implemented.
2. **SSE round-trip under live MongoDB** — requires a live replica set and browser. The `watch_qsos`

single trusted callsign configured by the operator. This is architecturally correct for the stated
use case and was a deliberate Phase 17 decision.

## Notes and Observations

1. **`import_qsos_from_bytes` is not used by the UDP path** — this is expected and correct.
   Phase 17 uses `build_qso_dict` + `find_duplicate` + `QSO.insert` directly to enable the
   `profile=user` auto-stamping parameter, which `import_qsos_from_bytes` does not expose.
   Both paths call the same underlying functions so processing is semantically identical.

2. **Lazy imports in `_handle_datagram`** — `parse_adi`, `QSO`, `_REQUIRED_FIELDS`,
   `build_qso_dict`, and `find_duplicate` are all imported inside the function body. This pattern
   correctly avoids circular imports at module load time and matches the pattern in
   `import_qsos_from_bytes`. The patch targets in tests correctly reflect the resolved namespaces
   (`app.qso.service.find_duplicate`, `app.qso.models.QSO`, `app.adif.parser.parse_adi`).

3. **`_background_tasks` GC safety** — the set is populated in `datagram_received` via
   `task.add_done_callback(self._background_tasks.discard)`. This prevents asyncio Tasks from
   being garbage-collected while in-flight, required since Python 3.12.

4. **Docker comment** — `docker-compose.yml` documents that `UDP_BIND_HOST=0.0.0.0` is required
   for host-to-container datagram delivery. Without this, the default `127.0.0.1` would reject
   datagrams arriving through the Docker network bridge.

5. **`populate_by_name=True` on QSO model** — confirms that `QSO(**qso_dict)` with
   `operator_callsign="VK2ABC"` constructs correctly despite the field having `alias="_operator"`.
