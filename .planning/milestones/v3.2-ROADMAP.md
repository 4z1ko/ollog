# Milestone v3.2: ACLog Full-Record Import

**Status:** ✅ SHIPPED 2026-06-09
**Phases:** 63
**Total Plans:** 1

## Overview

Improve the ACLog bridge so imported QSOs can include the full ACLog record, including user-customized Other fields, by using ACLog's `INCLUDEALL` API flow rather than relying only on the minimal `ENTEREVENT` payload.

This milestone enriched the existing ACLog TCP bridge while preserving its current safety behavior. After ACLog emits a saved-QSO `ENTEREVENT`, ollog now requests a full `LIST INCLUDEALL` record, correlates it to the pending event, merges full-record fields with cached live state, and falls back to the original event path when enrichment is unavailable or unsafe.

## Milestone Summary

**Key Decisions:**

- Use ACLog `LIST INCLUDEALL` as an enrichment request after `ENTEREVENT`, without blocking the socket loop indefinitely.
- Treat `CALL`, `QSO_DATE`, and `TIME_ON` as required stable identity fields for full-record correlation, with optional band/mode compatibility checks when present.
- Preserve all non-empty safe ADIF-like fields from full-record responses instead of hard-coding only a small ACLog field subset.
- Keep `OTHER_1` through `OTHER_8` preserved by default and map them to configured Custom QSO Fields only after merge.
- Keep existing `ENTEREVENT` plus live textbox update behavior as the fallback path for old ACLog versions, missing responses, mismatches, or reconnects.

**Issues Resolved:**

- ACLog bridge imports no longer depend only on the minimal `ENTEREVENT` payload when ACLog exposes richer saved-record data.
- ACLog Other fields can now flow into ollog imports through full-record data or cached live state.
- Bridge enrichment behavior has deterministic parser/client tests that do not require a live ACLog instance.

**Issues Deferred:**

- Live ACLog end-to-end verification remains manual because this environment does not provide a live ACLog TCP API server.
- `pytest` execution remains blocked in this shell because `uv`, `pytest`, and a Python environment with pytest are unavailable; syntax compilation passed and deterministic tests were added for CI/developer environments.
- No milestone audit was run because `gsd-sdk` is not available in this environment; Phase 63 UAT and verification artifacts passed.

**Technical Debt Incurred:**

- Full-record response support is intentionally conservative and fixture-driven; if future ACLog versions emit a different wrapper shape, parser fixtures may need to expand.
