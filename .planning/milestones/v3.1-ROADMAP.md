# Milestone v3.1: Per-User QSO Collections

**Status:** ✅ SHIPPED 2026-06-08
**Phases:** 59-62
**Total Plans:** 4

## Overview

Store each user's QSO records in a dedicated MongoDB collection named `<username>_qsos` while preserving all existing operator-facing behavior.

This milestone converted QSO storage from one shared physical collection into username-derived per-user collections. The refactor preserved the public API, browser workflows, ADIF shape, duplicate behavior, UDP/API-token routing, stats, admin clear-log, live feed behavior, and full-database backup/restore.

## Milestone Summary

**Key Decisions:**

- Collection names are derived from `User.username`, not callsign or request data.
- The QSO document keeps `_operator` as callsign for ADIF/profile/display semantics while physical routing uses username.
- The migration is copy-only and idempotent; unresolved or ambiguous ownership is reported, never guessed.
- The legacy shared `qsos` collection remains for migration/backward-compatible helpers; runtime user workflows route to per-user collections.
- Live feed now broadcasts from successful app write paths instead of watching the legacy shared collection at startup.
- Backup/restore remains full-database scoped so dynamically named collections are included naturally.

**Issues Resolved:**

- Eliminated shared-collection routing from direct QSO CRUD, import/export, UDP/API-token, stats, admin clear-log, and live-feed runtime paths.
- Added migration coverage for existing shared data, including soft-deleted rows, rowHash values, ADIF extras, and unresolved ownership reporting.
- Added regression coverage for representative cross-feature isolation and compatibility paths.

**Issues Deferred:**

- Optional admin UI to inspect migration status and unresolved legacy operators.
- Optional username rename tooling with explicit collection rename/relink behavior.
- Docker end-to-end verification remains pending because this environment does not provide a live Docker/Mongo production-like stack.

**Technical Debt Incurred:**

- Legacy Beanie `QSO.Settings.name = "qsos"` remains as a validation/serialization and migration compatibility shape.
- `QSO.get_pymongo_collection()` still appears in legacy startup migrations and rowHash backfill helpers.
- Some Mongo-backed integration tests skip when local MongoDB is unavailable; fake/unit coverage protects the highest-risk behavior in this sandbox.
