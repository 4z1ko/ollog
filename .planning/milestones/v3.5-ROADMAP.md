# Milestone v3.5: ACLog Registered Operator Routing

**Status:** SHIPPED 2026-06-16
**Phases:** 66
**Total Plans:** 1

## Overview

When multiple ollog operators point ACLog bridges at the same remote ACLog computer, ollog imports only QSOs whose ACLog record-level operator identity matches the ollog operator, and skips/reports records with missing or unmatched operator identity.

## Phases

### Phase 66: ACLog Operator Identity Routing

**Goal:** Filter live ACLog bridge imports and manual ACLog sync records by ACLog record-level operator identity so two ollog operators can safely share one remote ACLog API endpoint.
**Depends on:** Phase 63 (ACLog full-record parsing), Phase 64 (manual ACLog sync), Phase 61/62 (per-user QSO collection routing), Phase 57 (rowHash duplicate behavior)
**Plans:** 1 plan

Plans:

- [x] 66-01-PLAN.md - ACLog operator identity extraction, sync/live filtering, reports, tests, and docs.

**Requirements:** ACOP-01, ACOP-02, ACOP-03, ACOP-04, ACOP-05, ACOP-06, ACOP-07, ACOP-08, ACOP-09

**Success Criteria:**

1. ollog extracts a normalized ACLog operator identity from verified full-record fields such as `OPERATOR`.
2. Manual sync imports only records whose ACLog identity matches the authenticated ollog operator and skips missing/unmatched identities.
3. Live ACLog bridge ingestion imports only records whose ACLog identity matches the bridge-owning ollog operator and skips missing/unmatched identities.
4. Skipped missing/unmatched operator records are counted and reported without being inserted into any per-user QSO collection.
5. Two operators configured against the same ACLog host/port can sync or bridge without cross-importing each other's QSOs.
6. Matching records keep existing full-record field preservation, Other/custom-field mapping, duplicate detection, rowHash handling, and `<username>_qsos` collection routing.
7. Focused tests cover identity extraction, sync filtering, live bridge filtering, skip reports, and shared-remote two-operator behavior.
8. Operator documentation explains the shared ACLog remote setup and skip behavior for missing/unmatched operator identity.

## Milestone Summary

**Key Decisions:**

- Use ACLog `OPERATOR` as the only authoritative record-level identity field for v3.5.
- Do not use `STATION_CALLSIGN`, `MY_CALL`, or similar station-like fields as fallback authorization fields.
- Block live bridge `ENTEREVENT` fallback imports when no matching full record with operator identity is available.
- Report missing and unmatched operator records using counts and bounded examples, without exposing full skipped records.

**Key Accomplishments:**

- Added `app/aclog/identity.py` for normalized ACLog operator identity matching.
- Updated manual ACLog sync to skip missing/unmatched `OPERATOR` records before custom-field mapping or QSO ingest.
- Updated live ACLog bridge ingestion to skip unmatched full records and identity-less fallback events.
- Added Profile Settings sync report counts for missing and unmatched ACLog operator records.
- Documented shared remote ACLog computer behavior and troubleshooting.
- Added focused identity, manual sync, live bridge, profile report, UAT, security, validation, and audit artifacts.
