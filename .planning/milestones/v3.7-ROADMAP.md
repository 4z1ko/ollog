## Overview

Milestone v3.7 gave administrators explicit control over the Application Logs Recent Logs table by adding a current-browser Pause/Start live-feed control and a confirmation-gated Clear Log Messages action. The milestone preserves application log settings, retention behavior, future logging, and all QSO/operator data.

## Phases

### Phase 70: Admin Application Log Controls

**Goal:** Add pause/resume and clear controls to the admin Recent Logs table while preserving logging settings, future log capture, and QSO data.
**Depends on:** Phase 67, Phase 68, Phase 69
**Plans:** 1 plan

Plans:

- [x] 70-01: Admin Application Log Controls

**Details:**

- Added a LIVE/PAUSED/OFFLINE status badge, Pause/Start button, and Clear Log Messages button to the Recent Logs card header.
- Pause suppresses automatic SSE row insertion and near-live polling only in the current browser tab.
- Start immediately refreshes/reconciles recent rows and resumes automatic live updates.
- Existing filters, reset, Previous, and Next remain intentional refresh paths while paused.
- Clear Log Messages opens a confirmation modal before deletion.
- Confirmed clear deletes only `ApplicationLog` records, preserves `ApplicationLogSettings` and unrelated collections, then attempts a forced `application_logs_cleared` audit row.
- If the audit row cannot be written after clear, the user sees a clear-success note instead of a failed destructive operation.
- Admin documentation and focused tests cover pause/resume, clear scope, audit continuity, and safety guarantees.

## Milestone Summary

**Key Decisions:**

- Pause/Start is a current-browser control only; it does not pause server-side logging, broadcasts, storage, or other browser sessions.
- Clear Log Messages clears all stored application log records, not just the visible or filtered rows.
- The clear action deletes first and writes a forced audit event afterward so the audit row is not removed by the clear operation.
- Existing admin modal and HTMX fragment patterns were reused instead of introducing a new UI framework or page redesign.

**Issues Resolved:**

- Added admin control over Recent Logs live updates without changing internal logging behavior.
- Added a confirmation-gated database clear path for application logs.
- Preserved explicit filter/pagination refresh behavior while automatic live updates are paused.

**Issues Deferred:**

- Export/download filtered application logs.
- Clear by date range or active filters.
- Password-confirmed clear behavior.
- Global admin control to pause all live log streams.
- Cross-node coordination for live log stream controls in multi-instance deployments.

- None identified by milestone audit.
