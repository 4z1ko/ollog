# Milestone v2.0: Database Backup

**Status:** ✅ SHIPPED 2026-04-14
**Phases:** 37–38
**Total Plans:** 2

## Overview

Give the admin a one-click way to download a full MongoDB database backup with a timestamped filename. The existing `run_backup()` engine (shipped in v1.8) was wired to a new admin UI download endpoint with cookie auth, asyncio event-loop safety, and a plain browser anchor for the download trigger.

## Milestone Summary

**Key Decisions:**

- Sync/async split in `dump.py`: `asyncio.to_thread` requires a sync callable — passing `async def run_backup` would silently return a coroutine object instead of a Path; extracted `_write_backup` sync helper resolves this
- `require_admin_cookie` (not `require_admin`) on both routes: browser sends cookie not Bearer header; wrong dependency causes silent 302 redirect to login masking the failure
- Plain `<a href>` anchor (not `hx-get`): HTMX intercepts XHR responses and silently discards `Content-Disposition: attachment` binary payloads — download never fires with any `hx-*` attribute
- No `npm run build` required: all CSS tokens used in `backup.html` (`.card`, `.btn-primary`, `.nav-item`, `.nav-item-active`) already compiled in `output.css` from v1.9
- Per-page sidebar pattern: each admin template overrides `{% block sidebar_nav %}` entirely with both nav links at correct active states — no conditional logic in `base_app.html`

**Issues Resolved:**

- BACK-05: `datetime.utcnow()` Python 3.12 deprecation removed in Phase 37
- BACK-04 filename format: plan checker caught planner using `%Y-%m-%dT%H-%M-%SZ` (mismatched REQUIREMENTS.md canonical example); corrected to `%Y-%m-%d-%H-%M-%S`

**Technical Debt Incurred:**

- Pre-existing: `tests/test_qso_schema.py::test_qso_duplicate_rejected` and all MongoDB integration tests require a live `mongodb:27017` (Docker-internal) — cannot run on host without Docker. Pre-existed before v2.0; no regressions introduced.
