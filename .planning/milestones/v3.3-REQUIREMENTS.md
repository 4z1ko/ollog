# Requirements: ollog — v3.3 ACLog QSO Sync

**Defined:** 2026-06-12
**Core Value:** Multiple operators can log QSOs simultaneously under their own callsigns without conflicts or data loss.

## v3.3 Requirements

### ACLog Sync

- [x] **ACSYNC-01**: Operator can start a manual sync for each configured ACLog bridge from the Profile Settings page.
- [x] **ACSYNC-02**: Sync connects to the selected ACLog bridge and requests all remote QSOs with `<CMD><LIST><INCLUDEALL></CMD>`.
- [x] **ACSYNC-03**: Sync parses the full-record ACLog response and converts every returned non-empty safe ADIF-like field into the ollog QSO document shape.
- [x] **ACSYNC-04**: Sync preserves ACLog Other fields and applies the existing custom QSO field mapping behavior where configured.
- [x] **ACSYNC-05**: Sync inserts only QSOs missing from the authenticated operator's `<username>_qsos` collection and skips QSOs already present locally.
- [x] **ACSYNC-06**: Sync uses existing duplicate/rowHash behavior so duplicate detection remains consistent with browser, REST, ADIF import, UDP, and live ACLog ingestion paths.
- [x] **ACSYNC-07**: Operator receives an inline completion report showing how many remote QSOs were missing/imported, how many were already present/skipped, and any connection or parse errors.
- [x] **ACSYNC-08**: Sync action is only available for the logged-in operator's own configured bridges and cannot target another operator's collection.
- [x] **ACSYNC-09**: Existing ACLog live bridge import behavior, profile saving, bridge configuration, and Custom QSO Fields continue to work unchanged.

## Future Requirements

### ACLog Sync Enhancements

- **ACSYNC-FUT-01**: Operator can run scheduled/background ACLog sync without pressing a button.
- **ACSYNC-FUT-02**: Operator can preview remote-only QSOs before importing them.
- **ACSYNC-FUT-03**: Operator can choose a date range or maximum record count for ACLog sync.
- **ACSYNC-FUT-04**: Admin can view sync history or diagnostics across operators.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Automatic scheduled sync | Current milestone is manual operator-triggered sync only. |
| Updating existing local QSOs from ACLog | Requirement says only add non-existing QSOs; overwriting local records risks data loss and needs separate review semantics. |
| Deleting local QSOs missing from ACLog | Sync is additive only; local log remains authoritative for existing records. |
| ACLog ADIF export automation | ACLog API returns XML-like LIST records, not an all-log ADIF response; ollog will parse API records directly. |
| Cross-operator bridge sync | Each operator syncs only their own configured bridges into their own username-derived collection. |
