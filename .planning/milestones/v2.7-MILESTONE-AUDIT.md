### Integration Finding: aria-label Semantics (Non-Breaking)

The padlock buttons' `aria-label` strings describe current state rather than the action they will take:
- When locked: `aria-label="Lock date field"` — should be `"Unlock date field"` (the action)
- When unlocked: `aria-label="Unlock date field"` — should be `"Lock date field"` (the action)

**Affected REQ-IDs:** DATE-03, TIME-03 (at accessibility/screen-reader level only)
**Functional impact:** None. The lock/unlock behavior, SVG icon swap, and visual state are all correct. This is a screen-reader-only issue.
