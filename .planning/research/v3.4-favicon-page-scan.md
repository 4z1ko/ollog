# v3.4 Favicon Page Scan

**Date:** 2026-06-13
**Milestone:** v3.4 Responsive Favicon

## Asset Source

The favicon bundle exists under `favicon/`:

- `favicon/favicon.ico`
- `favicon/favicon-16x16.png`
- `favicon/favicon-32x32.png`
- `favicon/apple-touch-icon.png`
- `favicon/android-chrome-192x192.png`
- `favicon/android-chrome-512x512.png`
- `favicon/site.webmanifest`

The user specified `favicon/favicon.ico` as the favicon basis for this milestone.

## Template Surface

Full-page templates:

- `templates/log/login.html` extends `templates/base.html`
- `templates/admin/login.html` extends `templates/base.html`
- `templates/base_app.html` extends `templates/base.html`
- Operator app pages extending `base_app.html`: `about.html`, `form.html`, `import.html`, `log.html`, `profile.html`, `stats.html`
- Admin app pages extending `base_app.html`: `backup.html`, `restore.html`, `users.html`

Partial templates do not own a `<head>` and should inherit favicon behavior through their host page:

- Operator partials: ACLog sync result, clear-log modal/success, feed rows, import report, log table, profile result, QSO result/rows/edit, token fragments
- Admin partials: clear-log modal/success, restore modal fragments, users table

## Static Serving

- Operator app (`app/main.py`) mounts `/static` from `static/`.
- Admin app (`app/admin_main.py`) also mounts `/static` from `static/`.
- The favicon bundle currently lives outside `static/`, so implementation should either copy the selected favicon files into `static/` or explicitly mount the `favicon/` directory for both apps.

## Recommended Implementation

- Prefer app-owned static paths such as `/static/favicon/favicon.ico` so both operator and admin apps use their existing `/static` mounts.
- Add favicon link tags once in `templates/base.html` so every full-page operator/admin template inherits them.
- If using the broader bundle, include compatible links for `.ico`, 16/32 PNG icons, Apple touch icon, and web manifest, but keep `favicon/favicon.ico` as the primary source.
- Verify no page-specific template needs a separate favicon because all full pages inherit from the shared base chain.
