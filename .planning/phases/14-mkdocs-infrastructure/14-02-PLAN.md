  <name>Task 1: Mount site/ at /guide in FastAPI and update Dockerfile</name>
  <files>app/main.py, Dockerfile</files>
  <action>
1. In `app/main.py`, add a StaticFiles mount for `/guide` BEFORE the existing `/static` mount (line 115). Mount order is load-bearing in FastAPI — mounts registered later take precedence for overlapping paths, so `/guide` must come first.

   Add this line before `app.mount("/static", ...)`:
   ```python
   app.mount("/guide", StaticFiles(directory="site", html=True), name="guide")
   ```

   The `html=True` parameter is critical — it enables automatic `index.html` serving so that `/guide` serves `site/index.html` and `/guide/somepage/` serves `site/somepage/index.html`.

2. In `Dockerfile`, add `COPY site/ site/` after the existing COPY lines (after `COPY static/ static/` on line 12). This puts the pre-built docs into the production image without needing MkDocs installed.

   The Dockerfile COPY section should look like:
   ```dockerfile
   COPY app/ app/
   COPY templates/ templates/
   COPY static/ static/
   COPY site/ site/
   ```

3. Verify the mount works by starting the dev server and checking /guide:
   - Start: `uv run uvicorn app.main:app --port 8765 &` (use non-standard port to avoid conflicts)
   - Test index: `curl -s -o /dev/null -w "%{http_code}" http://localhost:8765/guide`  — expect 200
   - Test CSS asset: `curl -s -o /dev/null -w "%{http_code}" http://localhost:8765/guide/assets/stylesheets/main.*.css` — expect 200 (glob the hash in filename)
   - Test JS asset: similarly check an assets/javascripts/ file returns 200
   - Kill the server after verification

   If assets return 404, check that `site_url` in mkdocs.yml ends with `/guide/` (trailing slash).
  </action>
