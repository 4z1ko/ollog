<task type="auto">
  <name>Task 1: Install mkdocs-material and create MkDocs configuration</name>
  <files>pyproject.toml, mkdocs.yml, docs/index.md</files>
  <action>
1. Run `uv add --dev "mkdocs-material==9.*"` to add mkdocs-material to `[dependency-groups].dev` in pyproject.toml. Verify it appears ONLY under `[dependency-groups].dev`, NOT in `[project].dependencies`.

2. Create `mkdocs.yml` at project root with this configuration:
   - `site_name: ollog` 
   - `site_url: http://localhost:8000/guide/` (trailing slash required — without it, MkDocs generates relative asset paths that break when served at a sub-path)
   - `use_directory_urls: true`
   - `theme:` with `name: material` and `palette:` with `scheme: slate` and `primary: indigo`
   - `nav:` with a single entry: `- Home: index.md`
   - No `docs_dir` or `site_dir` (defaults are correct: docs/ and site/)

3. Create `docs/index.md` with minimal scaffold content:
   ```markdown
   # ollog — Ham Radio Online Logbook

   Welcome to the ollog documentation.

   This site is under construction. Full content will be added in Phase 15.
   ```
  </action>
  <verify>
- `grep "mkdocs-material" pyproject.toml` shows the dependency under dev group
- `grep -c "mkdocs-material" pyproject.toml` returns exactly 1 (not duplicated into production deps)
- `cat mkdocs.yml` shows site_url ending in `/guide/`
- `cat docs/index.md` shows scaffold content
  </verify>
  <done>mkdocs-material is a dev-only dependency, mkdocs.yml configures Material theme with site_url pointing to /guide/, and docs/index.md exists as scaffold</done>
</task>

<task type="auto">
  <name>Task 2: Build site/ and commit it to the repo</name>
  <files>site/</files>
  <action>
1. Run `uv run mkdocs build --strict` — this must complete with exit code 0 and produce a `site/` directory. The `--strict` flag converts warnings to errors, so all nav entries must have corresponding files.

2. Verify site/ directory contains expected files:
   - `site/index.html` (the homepage)
   - `site/assets/` directory (CSS and JS from Material theme)
   - `site/sitemap.xml`

3. Check that `site/` is NOT in `.gitignore`. If it is, remove the `site/` entry — this project commits built docs to the repo per MKDOCS-05.

4. Stage and commit the entire `site/` directory with `git add site/` — this is intentional, the project does not use CI to build docs.

Note: Do NOT add `site/` to `.gitignore`. The design decision is to commit built docs so the Dockerfile can COPY them without installing MkDocs in the production image.
  </action>
  <verify>
- `uv run mkdocs build --strict` exits with code 0
- `ls site/index.html` exists
- `ls site/assets/` exists and contains CSS/JS files
- `git status` shows site/ files are tracked
  </verify>
  <done>site/ directory is built without errors and committed to the repository</done>
</task>
