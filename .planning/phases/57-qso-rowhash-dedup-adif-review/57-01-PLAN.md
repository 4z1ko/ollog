# Phase 57 Plan 01: QSO RowHash Deduplication and ADIF Duplicate Review

## Objective

Implement deterministic document-level deduplication for QSO documents and add an ADIF duplicate review flow that imports new QSOs automatically while letting the operator explicitly choose whether to import records that already exist.

## Acceptance Criteria

- `rowHash` is computed from canonical business values, excluding metadata and audit fields.
- MongoDB has a unique idempotent index on `rowHash`.
- REST, UI, UDP, and ADIF insert flows compute `rowHash` before insertion and report duplicates explicitly.
- QSO soft-delete updates `rowHash`.
- Existing records can be safely backfilled without overwriting existing hashes or deleting duplicate groups.
- ADIF import reports existing QSOs in a table with per-row checkboxes and a server-side revalidated import route.
- Focused tests, scoped lint, and compile validation pass.
