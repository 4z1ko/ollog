**# API Reference**

Brief intro: ollog exposes a REST API at the base URL (e.g., `http://localhost:8000`). Interactive Swagger docs are available at `/docs`. This page provides a complete reference with curl examples.

**## Authentication (DOCS-05)**

Two auth mechanisms:

**### Bearer Token (Primary)**
Most endpoints use Bearer token auth.
1. POST to /auth/token with form-encoded username + password
2. Extract `access_token` from JSON response
3. Send as `Authorization: Bearer <token>` header

```bash
TOKEN=$(curl -s -X POST http://localhost:8000/auth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=yourpassword" | jq -r .access_token)
```

Token expires after JWT_EXPIRE_MINUTES (default: 60 minutes).

**### HTTP-Only Cookie (SSE Only)**
The station feed endpoint (`GET /feed/station`) uses cookie authentication. Why: the browser's `EventSource` API cannot send custom headers like `Authorization`. Instead, the JWT is stored in an HttpOnly cookie during browser login. When using the API directly, most clients cannot easily obtain this cookie -- the station feed is designed primarily for browser consumption.

**## Auth Endpoints**

For each endpoint below, use this format:
- H3 heading with method and path
- One-line description
- **Auth:** what auth is required
- **Request:** body/params (if any)
- **Response:** status code and body shape
- **curl example**

### POST /auth/token
Obtain a JWT access token.
- Auth: None (this IS the login endpoint)
- Request: `application/x-www-form-urlencoded` with `username` and `password` fields
- Response: 200 `{"access_token": "...", "token_type": "bearer"}`
- Error: 401 if credentials invalid
- curl: (already shown above)

### GET /auth/me
Get the current authenticated user's info.
- Auth: Bearer token
- Response: 200 with user object (username, callsign, is_admin, enabled)
- Error: 401 if not authenticated
```bash
curl http://localhost:8000/auth/me \
  -H "Authorization: Bearer $TOKEN"
```

**## QSO Endpoints**

### POST /api/qsos/
Create a new QSO record.
- Auth: Bearer token
- Request: JSON body with fields: call (required), band, mode, qso_date (YYYYMMDD), time_on (HHMM or HHMMSS), rst_sent, rst_rcvd, and any additional ADIF fields (extra fields are accepted and stored)
- Response: 201 with QSO object including generated _id, auto-stamped _operator
- Error: 409 if duplicate detected (same CALL+BAND+MODE+operator within +/-2 min). Add `?force=true` query param to override.
- Error: 422 if validation fails
```bash
curl -X POST http://localhost:8000/api/qsos/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"call": "DL1ABC", "band": "20m", "mode": "SSB", "qso_date": "20240415", "time_on": "1430", "rst_sent": "59", "rst_rcvd": "59"}'
```

### GET /api/qsos/
List QSOs for the authenticated operator.
- Auth: Bearer token
- Response: 200 with `{"items": [...], "total": N}` -- only returns QSOs where _operator matches the authenticated user. Soft-deleted QSOs excluded.
```bash
curl http://localhost:8000/api/qsos/ \
  -H "Authorization: Bearer $TOKEN"
```

### GET /api/qsos/{id}
Get a single QSO by ID.
- Auth: Bearer token
- Response: 200 with QSO object
- Error: 404 if not found or belongs to different operator
```bash
curl http://localhost:8000/api/qsos/66123abc... \
  -H "Authorization: Bearer $TOKEN"
```

### PATCH /api/qsos/{id}
Update a QSO record.
- Auth: Bearer token
- Request: JSON body with fields to update (partial update)
- Response: 200 with updated QSO object
- Error: 404 if not found or belongs to different operator
```bash
curl -X PATCH http://localhost:8000/api/qsos/66123abc... \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"rst_sent": "57"}'
```

### DELETE /api/qsos/{id}
Soft-delete a QSO (sets _deleted=true, not physically removed).
- Auth: Bearer token
- Response: 204 No Content
- Error: 404 if not found or belongs to different operator
```bash
curl -X DELETE http://localhost:8000/api/qsos/66123abc... \
  -H "Authorization: Bearer $TOKEN"
```

**## ADIF Endpoints**

### POST /api/adif/import
Import QSOs from an ADIF (.adi) file.
- Auth: Bearer token
- Request: multipart/form-data with `file` field (max 10 MB)
- Response: 200 with ADIFImportReport: `{"created": N, "duplicates": N, "errors": [...]}`. Each error has `record` (int), `call` (string or null), and `reason` (string).
- Duplicate detection: same CALL+BAND+MODE+operator within +/-2 min window
- Note: Import does NOT auto-stamp OPERATOR or STATION_CALLSIGN -- file values are preserved (for historical record fidelity)
```bash
curl -X POST http://localhost:8000/api/adif/import \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@mylog.adi"
```

### GET /api/adif/export
Export the authenticated operator's log as an ADIF 3.1.4 file.
- Auth: Bearer token
- Response: 200 with `text/plain` streaming body (content-disposition: attachment)
- Only exports the authenticated operator's QSOs. Soft-deleted records excluded.
```bash
curl http://localhost:8000/api/adif/export \
  -H "Authorization: Bearer $TOKEN" \
  -o mylog.adi
```

**## Profile Endpoints**

### GET /api/profile/
Get the authenticated operator's profile.
- Auth: Bearer token
- Response: 200 with profile object (station_callsign, grid_square, name, etc.)
```bash
curl http://localhost:8000/api/profile/ \
  -H "Authorization: Bearer $TOKEN"
```

### PATCH /api/profile/
Update the authenticated operator's profile.
- Auth: Bearer token
- Request: JSON body with fields to update (partial update). Accepted fields: station_callsign, grid_square, name.
- Response: 200 with updated profile object
- Note: Setting station_callsign causes it to be auto-stamped on future QSOs as STATION_CALLSIGN
```bash
curl -X PATCH http://localhost:8000/api/profile/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"station_callsign": "W1AW", "grid_square": "FN31pr"}'
```

**## Admin Endpoints**

All admin endpoints require Bearer token from an admin account.

### GET /admin/users/
List all user accounts.
- Auth: Bearer token (admin)
- Response: 200 with array of user objects
```bash
curl http://localhost:8000/admin/users/ \
  -H "Authorization: Bearer $TOKEN"
```

### POST /admin/users/
Create a new operator account.
- Auth: Bearer token (admin)
- Request: JSON with username, password, callsign (required). Optional: is_admin (bool, default false)
- Response: 201 with created user object
```bash
curl -X POST http://localhost:8000/admin/users/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"username": "op1", "password": "securepass", "callsign": "W1AW"}'
```

### PATCH /admin/users/{username}/enabled
Enable or disable an operator account.
- Auth: Bearer token (admin)
- Request: JSON with `enabled` (bool)
- Response: 200 with updated user object
- Error: 409 if trying to disable the last enabled admin (lockout guard)
```bash
curl -X PATCH http://localhost:8000/admin/users/op1/enabled \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```

### POST /admin/users/{username}/reset-password
Reset a user's password.
- Auth: Bearer token (admin)
- Request: JSON with `new_password` (string)
- Response: 200
- Note: This is POST, not PATCH
```bash
curl -X POST http://localhost:8000/admin/users/op1/reset-password \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"new_password": "newpass123"}'
```

**## SSE Endpoint**

### GET /feed/station
Server-Sent Events stream of new QSOs across all operators.
- Auth: HTTP-only cookie (NOT Bearer token -- see Authentication section above)
- Response: text/event-stream. Each event is a JSON QSO object.
- Note: This endpoint is excluded from the OpenAPI schema (/docs) because it uses cookie auth and cannot be exercised from Swagger UI.
```bash
# Browser: works automatically via cookie set at login
# curl (requires cookie from browser session):
curl -N --cookie "session=YOUR_COOKIE" http://localhost:8000/feed/station
```
