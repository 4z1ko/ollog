**# Deployment**

**## Prerequisites**
- Docker and Docker Compose v2 installed
- A machine or VM with port 8000 available
- Note: MongoDB is included in the compose stack, no separate install needed

**## Quick Start**
Step-by-step:
1. Clone the repo
2. Create `.env` file (show template below)
3. `docker compose up -d`
4. Verify: `curl http://localhost:8000/auth/me` should return 401 (confirms API is up)
5. Verify docs: open `http://localhost:8000/guide` in browser

**## Environment Variables**
Table format with these exact vars (sourced from app/config.py and docker-compose.yml):

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| SECRET_KEY | Yes | (none) | JWT signing key. MUST set a strong random value. If changed, all existing tokens invalidate. |
| MONGODB_URI | No | mongodb://mongodb:27017/?replicaSet=rs0 | MongoDB connection string. The docker-compose.yml overrides the code default to include replicaSet=rs0. |
| MONGODB_DB | No | ollog | Database name |
| JWT_EXPIRE_MINUTES | No | 60 | Token lifetime in minutes |
| ADMIN_USERNAME | No | (none) | Bootstrap admin username (one-time, first startup only) |
| ADMIN_PASSWORD | No | (none) | Bootstrap admin password (one-time, first startup only) |
| ADMIN_CALLSIGN | No | (none) | Bootstrap admin callsign (one-time, first startup only) |

Show a sample `.env` file:
```
SECRET_KEY=change-me-to-a-long-random-string
ADMIN_USERNAME=admin
ADMIN_PASSWORD=changeme123
ADMIN_CALLSIGN=N0CALL
```

**## Bootstrap Admin Account**
Explain one-time behavior: On first startup, if ADMIN_USERNAME, ADMIN_PASSWORD, and ADMIN_CALLSIGN are all set, the app creates an admin account. This only happens once -- if the admin already exists, the vars are ignored. After first startup you can remove them from .env.

**## Verification Steps**
Numbered checklist:
1. `docker compose ps` -- both services running and healthy
2. `curl http://localhost:8000/auth/me` -- returns 401 (API responding)
3. Login: `curl -X POST http://localhost:8000/auth/token -H "Content-Type: application/x-www-form-urlencoded" -d "username=admin&password=changeme123"` -- returns JSON with access_token
4. Open `http://localhost:8000/guide` -- documentation site loads
5. Open `http://localhost:8000/docs` -- Swagger UI loads

**## Updating**
Brief: `git pull && docker compose build && docker compose up -d`. Note: `site/` is pre-built in the image, no MkDocs needed at runtime.

**## MongoDB Replica Set**
Explain why replica set is required: ollog uses MongoDB change streams for the live station feed (SSE). Change streams require a replica set. The docker-compose.yml starts MongoDB with `--replSet rs0` and auto-initializes via healthcheck. If using an external MongoDB, ensure it runs as a replica set.

**# Admin Guide**

Brief intro: Admin accounts can create operator accounts, enable/disable them, and reset passwords. All admin endpoints require a valid Bearer token from an admin account.

**## Authentication**
Show how to get a token:
```bash
TOKEN=$(curl -s -X POST http://localhost:8000/auth/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=yourpassword" | jq -r .access_token)
```

**## List Users**
```bash
curl http://localhost:8000/admin/users/ \
  -H "Authorization: Bearer $TOKEN"
```
Describe response: array of user objects with username, callsign, is_admin, enabled fields.

**## Create an Operator Account**
```bash
curl -X POST http://localhost:8000/admin/users/ \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"username": "op1", "password": "securepass", "callsign": "W1AW"}'
```
Returns 201 with the created user object. Note: callsign is uppercased automatically.

**## Enable / Disable an Account**
```bash
curl -X PATCH http://localhost:8000/admin/users/op1/enabled \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"enabled": false}'
```
Explain: Disabling prevents login. A disabled user's existing token still works until it expires (JWT_EXPIRE_MINUTES).

**### Last-Admin Lockout Guard**
Explain: The API prevents disabling the last remaining enabled admin account. If you try, you get a 409 Conflict response. This ensures you can never lock yourself out entirely.

**## Reset a User's Password**
```bash
curl -X POST http://localhost:8000/admin/users/op1/reset-password \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"new_password": "newpass123"}'
```
Note: This is a POST endpoint (not PATCH). The user's existing tokens remain valid until they expire.

Use the same plain, direct tone. Each section should be self-contained -- an admin can jump to the section they need.
