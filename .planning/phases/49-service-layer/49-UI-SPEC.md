## Template Change Contract

This section replaces the standard "Copywriting Contract" for this backend-only phase. It is the
sole UI deliverable.

### SSE Auto-Refresh Sentinel — `templates/log/log_table.html` line 1

**Current condition (before Phase 49):**
```jinja2
{% if page == 1 and sort == '-qso_date_utc' and not filters.call and not filters.band and not filters.mode and not filters.date_from and not filters.date_to %}
```

**Required condition (after Phase 49):**
```jinja2
{% if page == 1 and (sort == '-qso_date_utc' or sort == '-_created_at') and not filters.call and not filters.band and not filters.mode and not filters.date_from and not filters.date_to %}
```

**Contract rules:**
- The `or sort == '-_created_at'` clause is appended inside the existing `sort == '-qso_date_utc'`
  position, wrapped in parentheses to preserve operator precedence with the `and not filters.*` chain.
- No HTML structure change — the `<span id="auto-refresh-ok" hidden></span>` element is unchanged.
- No new CSS classes are introduced.
- The `hidden` attribute behavior is unchanged: sentinel appears in DOM only when condition is true,
  triggering the SSE feed's JavaScript listener.
- This change must be applied before or alongside the `_ALLOWED_SORT_FIELDS` constant in
  `app/qso/service.py` — if `-_created_at` is not in the allowlist, the sentinel branch is
  unreachable and the condition extension is inert.

**Interaction contract:** When a user loads the log table sorted by `-_created_at` (newest-entered
first) on page 1 with no filters active, SSE auto-refresh is enabled — new QSOs submitted by any
operator appear in the table without a manual reload. This matches the existing behavior for
`-qso_date_utc` sort.

**Verification:** After implementation, confirm via browser devtools that `#auto-refresh-ok` is
present in the DOM when sort is `-_created_at`, page is 1, and no filters are active; and absent
when any filter is set or page > 1.
