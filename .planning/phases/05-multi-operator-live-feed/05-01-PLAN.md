  <files>docker-compose.yml</files>
  <action>
Update the `mongodb` service in docker-compose.yml to run as a single-node replica set:

1. Add `command: ["--replSet", "rs0", "--bind_ip_all", "--port", "27017"]` to the mongodb service.

2. Replace the existing healthcheck with:
```yaml
healthcheck:
  test: >
    echo "try { rs.status() } catch (err) {
      rs.initiate({_id:'rs0',members:[{_id:0,host:'mongodb:27017'}]})
    }" | mongosh --port 27017 --quiet
  interval: 5s
  timeout: 30s
  start_period: 0s
  retries: 30
```

3. Update the `MONGODB_URI` environment variable in the `api` service to include `?replicaSet=rs0`:
```
MONGODB_URI=mongodb://mongodb:27017/?replicaSet=rs0
```

Keep all other settings (ports, volumes, api service) unchanged.

Note: The `app/config.py` default for `mongodb_uri` is only used when no env var is set. The docker-compose.yml environment block overrides it. Do NOT change the default in config.py — local dev outside Docker can still connect to standalone MongoDB.
  </action>

  <name>Task 2: Concurrent write safety integration tests</name>

  <action>
Create `tests/test_concurrent_writes.py` with the following integration tests. All tests require MongoDB (use the socket-check skip pattern from conftest.py, or import `test_db` fixture if the existing one works — but note conftest.py's test_db only registers QSO, not User; create a local fixture if needed that registers both QSO and User models).

Each test should create operators and QSOs using direct Beanie model operations (not HTTP endpoints) to isolate the database behavior.

**Test 1: `test_two_operators_same_contact_both_succeed`**
- Create two QSOs with identical CALL="W1AW", BAND="20M", MODE="SSB", qso_date_utc=same timestamp, but different _operator values ("AA1AA" and "BB2BB").
- Insert both via `QSO(**qso_dict).insert()`.
- Assert both inserts succeed (no DuplicateKeyError).
- Query the collection and assert exactly 2 documents exist.
- Assert each document has the correct _operator value.

**Test 2: `test_concurrent_inserts_no_lost_writes`**
- Use `asyncio.gather()` to run 20 concurrent QSO inserts: 10 from operator "AA1AA" and 10 from operator "BB2BB". Each QSO should have a unique CALL (e.g., "W{i}TEST") so no duplicate detection issues.
- After all 20 inserts complete, count documents in the collection.
- Assert exactly 20 documents exist.
- Assert 10 belong to "AA1AA" and 10 to "BB2BB".

**Test 3: `test_concurrent_inserts_correct_attribution`**
- Use `asyncio.gather()` to insert 5 QSOs from "AA1AA" (CALLs: "W1A", "W2A", ..., "W5A") and 5 from "BB2BB" (CALLs: "W1B", "W2B", ..., "W5B") concurrently.
- After completion, for each operator query their QSOs and assert the CALL values match exactly what was submitted (no cross-contamination).

**Test 4: `test_same_operator_duplicate_race_documented`**
- Insert two identical QSOs (same _operator, CALL, BAND, MODE, qso_date_utc) concurrently via `asyncio.gather()`.
- Assert both succeed (since unique index was removed in phase 03-02).
- Add a docstring documenting: "This test proves the known race window — two identical QSOs from the same operator can both insert if concurrent. The app-level find_duplicate() does not protect against sub-millisecond races. This is accepted per design decision in 03-02."
- Count documents and assert exactly 2 exist (proving the race window exists and is accepted).

Use `@pytest.mark.asyncio` decorator on all tests. Use the helper pattern from existing tests:
```python
from datetime import datetime, timezone
from app.qso.models import QSO
```
