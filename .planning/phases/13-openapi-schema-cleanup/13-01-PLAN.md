  <files>app/qso/router.py</files>
  <action>
Add the following Pydantic models at the top of `app/qso/router.py` (after imports, before `_qso_to_dict`):

1. **QSOResponse** — mirrors what `_qso_to_dict()` returns:
   ```python
   class QSOResponse(BaseModel):
       model_config = ConfigDict(extra="ignore")
       id: str
       CALL: str
       BAND: Optional[str] = None
       MODE: Optional[str] = None
       qso_date_utc: Optional[str] = None
       operator_callsign: Optional[str] = None
       is_deleted: Optional[bool] = None
       FREQ: Optional[str] = None
       RST_SENT: Optional[str] = None
       RST_RCVD: Optional[str] = None
   ```
   Use `extra="ignore"` so arbitrary ADIF model_extra fields (QSO_DATE, TIME_ON, APP_* etc.) are silently dropped during response validation without causing 500 errors. The schema documents only the stable declared fields.

   NOTE: `_qso_to_dict()` uses `model_dump(by_alias=True)` so operator_callsign appears as `_operator` and is_deleted as `_deleted` in the output dict. The QSOResponse model must use `Field(alias="_operator")` for operator_callsign and `Field(alias="_deleted")` for is_deleted, with `model_config = ConfigDict(extra="ignore", populate_by_name=True)`.

   IMPORTANT: Verify this by reading what `_qso_to_dict()` actually returns. The function does `d = qso.model_dump(by_alias=True)` then sets `d["id"] = str(qso.id)` and pops `_id`. So the dict keys include `_operator`, `_deleted`, `CALL`, `BAND`, `MODE`, `qso_date_utc`, plus any model_extra keys. The QSOResponse must accept these aliased keys.

2. **QSOListResponse** — wrapper for list endpoint:
   ```python
   class QSOListResponse(BaseModel):
       items: list[QSOResponse]
       total: int
       page: int
       page_size: int
   ```

3. **DuplicateQSOError** — for 409 response documentation:
   ```python
   class DuplicateQSOError(BaseModel):
       duplicate: bool
       existing_id: str
       existing_call: str
       existing_band: str
       existing_mode: str
       existing_date: Optional[str] = None
   ```

Do NOT change `_qso_to_dict()` or `QSOCreateRequest` in this task — that is Task 2.
Do NOT change any endpoint function bodies — only models are added in this task.

<task type="auto">
  <name>Task 2: Annotate QSO decorators, add Field descriptions, expand delete docstring</name>
  <files>app/qso/router.py</files>
  <action>
Modify the five QSO endpoint decorators and the QSOCreateRequest model:

1. **POST `/`** — add `response_model=QSOResponse` and `responses` for 409:
   ```python
   @router.post(
       "/",
       status_code=status.HTTP_201_CREATED,
       response_model=QSOResponse,
       responses={
           409: {
               "model": DuplicateQSOError,
               "description": "Duplicate QSO detected within +/-2 min window. Use force=true to override.",
           }
       },
   )
   ```
   Also update the `force` parameter to include a description:
   ```python
   force: bool = Query(False, description="Override duplicate detection and force insert"),
   ```

2. **GET `/`** (list) — add `response_model=QSOListResponse`:
   ```python
   @router.get("/", status_code=status.HTTP_200_OK, response_model=QSOListResponse)
   ```

3. **GET `/{qso_id}`** — add `response_model=QSOResponse`:
   ```python
   @router.get("/{qso_id}", status_code=status.HTTP_200_OK, response_model=QSOResponse)
   ```

4. **PATCH `/{qso_id}`** — add `response_model=QSOResponse`:
   ```python
   @router.patch("/{qso_id}", status_code=status.HTTP_200_OK, response_model=QSOResponse)
   ```

5. **DELETE `/{qso_id}`** — no response_model (returns None/204). Update docstring to explicitly disclose soft-delete semantics per OAPI-06:
   ```python
   """Soft-delete a QSO by ID.

   The record is marked as deleted in MongoDB — it is NOT physically removed.
   Soft-deleted QSOs are excluded from all list, get, and export operations.
   To permanently remove data, direct database access is required.

   Returns 204 No Content on success. Returns 404 if not found, not owned,
   or already deleted.
   """
   ```

6. **QSOCreateRequest Field descriptions** — add `Field(description=...)` per OAPI-05:
   ```python
   CALL: str = Field(description="Callsign of the contacted station (e.g. W1AW)")
   QSO_DATE: str = Field(description="UTC date of QSO in YYYYMMDD format (e.g. 20240115)")
   TIME_ON: str = Field(description="UTC start time in HHMM or HHMMSS format (e.g. 1430 or 143045)")
   BAND: str = Field(description="Amateur band designator (e.g. 40m, 20m, 2m)")
   MODE: str = Field(description="Operating mode per ADIF spec (e.g. SSB, CW, FT8, FM)")
   FREQ: Optional[str] = Field(None, description="Frequency in MHz (e.g. 14.225)")
   RST_SENT: Optional[str] = Field(None, description="RST signal report sent to contacted station")
   RST_RCVD: Optional[str] = Field(None, description="RST signal report received from contacted station")
   ```
   Also add a note in the QSOCreateRequest docstring about OPERATOR vs STATION_CALLSIGN:
   ```python
   """Request model for QSO creation. extra='allow' captures arbitrary ADIF fields.

   OPERATOR and STATION_CALLSIGN are auto-stamped from the authenticated user's
   profile — do not include them in the request body. OPERATOR identifies the
   person at the key; STATION_CALLSIGN identifies the station's licensed callsign.
   """
   ```

   Import `Field` from pydantic if not already imported (it is already imported in the current file).
