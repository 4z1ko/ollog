# Plan 66-01: ACLog Operator Identity Routing

## Objective

Make ACLog live bridge ingestion and manual bridge sync safe when multiple ollog operators point at the same remote ACLog computer. ollog must import only records whose ACLog record-level operator identity matches the current ollog operator, and must skip/report records whose ACLog operator identity is missing, blank, or unmatched.

## Boundaries

In scope:

- Record-level ACLog operator identity extraction from parsed `LIST INCLUDEALL` records.
- Matching against the current bridge owner/authenticated operator with callsign normalization.
- Manual sync filtering and expanded inline report counts.
- Live bridge filtering after full-record enrichment and before QSO ingestion.
- Deterministic tests for two operators sharing one ACLog remote response.
- Operator documentation for shared remote ACLog setup and skip behavior.

Out of scope:

- Importing records with missing or unmatched ACLog operator identity.
- Cross-user dispatch where a bridge owned by operator A inserts into operator B's collection.
- Admin-managed ACLog label-to-ollog-user mappings.
- Updating existing local QSOs from ACLog.
- Scheduled/background sync.
- Changing ACLog itself or requiring live ACLog in automated tests.

## Threat Model

### Assets

- Operator-owned QSO records in `<username>_qsos`.
- Shared remote ACLog API responses that may contain multiple operators' QSOs.
- Operator profile callsign and bridge configuration.
- Manual sync report rendered into the authenticated Profile Settings page.

### Threats And Mitigations

- **T-66-01: Shared ACLog remote cross-imports another operator's QSOs.**  
  Mitigation: both manual sync and live bridge ingestion must gate records on matched ACLog record-level identity before insertion.

- **T-66-02: Missing identity silently falls back to bridge owner.**  
  Mitigation: missing identity is an explicit skip disposition; no event-only fallback may insert identity-less records.

- **T-66-03: Shared station callsign is mistaken for operator identity.**  
  Mitigation: prefer `OPERATOR`; treat station-like fields cautiously and test that shared station fields do not bypass the operator gate.

- **T-66-04: Report leaks another operator's full QSO details.**  
  Mitigation: skipped examples are bounded and minimal; do not render full records or all fields in the Profile Settings report.

- **T-66-05: Existing matching ACLog imports regress.**  
  Mitigation: matching records continue through existing parser, custom field mapping, duplicate/rowHash, and collection-routing paths; regression tests cover accepted and duplicate matches.
