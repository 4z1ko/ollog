  <name>Task 1: Log view UI routes (list, edit row, save, delete)</name>
  <files>app/qso/ui_router.py</files>
  <action>
Add the following endpoints to app/qso/ui_router.py (the file already exists from 03-03):

1. GET /log/view (response_class=HTMLResponse):
   - Requires auth: `callsign: str = Depends(get_current_operator_callsign_cookie)`.
   - Query params: page (int=1), page_size (int=50), call (Optional[str]=None), band (Optional[str]=None), mode (Optional[str]=None), date_from (Optional[str]=None), date_to (Optional[str]=None), sort (str="-qso_date_utc").
   - Parse date_from/date_to to UTC datetimes if provided (YYYYMMDD format).
   - Call get_qso_page(callsign, page, page_size, call, band, mode, date_from_dt, date_to_dt, sort).
   - Check HX-Request header: if HTMX request, return templates/log/log_table.html partial. Otherwise return templates/log/log.html (full page).
   - Template context: qsos (list), total, page, page_size, total_pages (ceil(total/page_size)), filters (call, band, mode, date_from, date_to), sort, callsign.

2. GET /log/qsos/{qso_id}/edit (response_class=HTMLResponse):
   - Requires auth.
   - Fetch QSO by ID. 404 if not found, not owned by operator, or deleted.
   - Return templates/log/qso_row_edit.html with the QSO data.
   - This replaces the view row with an editable row (HTMX outerHTML swap).

3. GET /log/qsos/{qso_id} (response_class=HTMLResponse):
   - Requires auth.
   - Fetch QSO by ID. 404 if not found, not owned, or deleted.
   - Return templates/log/qso_row.html — the view-mode row.
   - Used by "Cancel" button to restore the view row after edit mode.

4. PATCH /log/qsos/{qso_id} (response_class=HTMLResponse):
   - Requires auth.
   - Accept Form fields (all optional): CALL, QSO_DATE, TIME_ON, BAND, FREQ, MODE, RST_SENT, RST_RCVD.
   - Fetch QSO by ID. 404 if not found, not owned, or deleted.
   - Build update dict from non-empty form values.
   - Uppercase BAND and MODE if present.
   - If QSO_DATE or TIME_ON changed, recalculate qso_date_utc (use existing values for whichever field was not submitted).
   - Strip protected fields (_operator, _deleted, etc.).
   - Use `await qso.update({"$set": update_dict})` — raw $set with MongoDB field names.
   - Refetch the QSO and return templates/log/qso_row.html with updated data.

5. DELETE /log/qsos/{qso_id} (response_class=HTMLResponse):
   - Requires auth.
   - Fetch QSO by ID. 404 if not found, not owned, or already deleted.
   - Use `await qso.update({"$set": {"_deleted": True}})`.
   - Return empty string with 200 status (HTMX outerHTML swap removes the row).
   - NOTE: Return Response(content="", status_code=200) — HTMX replaces the <tr> with nothing, removing it from the table.

CRITICAL:
- All raw $set dicts use MongoDB alias: "_deleted" not "is_deleted", "_operator" not "operator_callsign".
- PATCH from form uses Form() parameters, not JSON body (HTMX hx-include sends form-encoded data).
- The PATCH endpoint must handle the case where hx-include="closest tr" sends ALL input values from the edit row.
- For the QSO ID parameter in routes, use PydanticObjectId from beanie if available, or str and convert with `PydanticObjectId(qso_id)` for the get() call.

**templates/log/qso_row.html** (HTMX partial — single <tr>):
```html
<tr id="qso-{{ qso.id }}">
  <td>{{ qso.qso_date_utc.strftime('%Y-%m-%d %H:%M') if qso.qso_date_utc else '' }} UTC</td>
  <td>{{ qso.CALL }}</td>
  <td>{{ qso.BAND or '' }}</td>
  <td>{{ qso.MODE or '' }}</td>
  <td>{{ qso.FREQ if qso.FREQ is defined else (qso.__dict__.get('FREQ', '') or '') }}</td>
  <td>{{ qso.RST_SENT if qso.RST_SENT is defined else '' }} / {{ qso.RST_RCVD if qso.RST_RCVD is defined else '' }}</td>
  <td class="actions">
    <button hx-get="/log/qsos/{{ qso.id }}/edit"
            hx-target="#qso-{{ qso.id }}"
            hx-swap="outerHTML">Edit</button>
    <button class="danger"
            hx-delete="/log/qsos/{{ qso.id }}"
            hx-target="#qso-{{ qso.id }}"
            hx-swap="outerHTML"
            hx-confirm="Delete this QSO? It can be recovered by an admin.">Delete</button>
  </td>
</tr>
```

NOTE on accessing model_extra fields (FREQ, RST_SENT, RST_RCVD): These are stored in QSO.model_extra, not as direct attributes. In Jinja2, access them via the QSO object. Since Beanie documents with extra="allow" store extra fields, they should be accessible as attributes. If not, the ui_router should convert QSOs to dicts before passing to templates. Test this during implementation — if `qso.FREQ` doesn't work in Jinja2, pass `qso.model_dump(by_alias=True)` as dict to the template instead.
