  <files>app/qso/ui_router.py, app/main.py</files>
  <action>
Create app/qso/ui_router.py following the pattern from app/admin/ui_router.py:

```python
"""QSO entry UI router — browser-based QSO logging for operators.

Serves HTML pages and HTMX partial responses for the QSO entry form.
All protected routes require cookie-based JWT auth.
Auth failures (401/403) are caught by the app-level exception handler
in main.py and redirected to /log/login.

Mounted at /log by app/main.py.
"""
```

Router: `ui_router = APIRouter(prefix="/log", tags=["log-ui"])`

Endpoints:

1. GET /log/login (response_class=HTMLResponse):
   - Render templates/log/login.html with {"error": None}.
   - Public — no auth required.

2. POST /log/login (response_class=HTMLResponse):
   - Accept Form fields: username, password.
   - Find user by username. If not found, disabled, or password wrong → re-render login.html with error.
   - NOTE: Unlike admin login, do NOT check role == "admin". Any enabled user (operator or admin) can log in here.
   - Create JWT token with sub, callsign, role.
   - Set HttpOnly cookie "access_token" and redirect to /log/.

3. GET /log/logout:
   - Delete cookie, redirect to /log/login.

4. GET /log/ (response_class=HTMLResponse):
   - Requires auth: `callsign: str = Depends(get_current_operator_callsign_cookie)`.
   - Render templates/log/form.html with {"callsign": callsign}.
   - This is the main QSO entry page.

5. POST /log/qsos (response_class=HTMLResponse):
   - Requires auth: `callsign: str = Depends(get_current_operator_callsign_cookie)`.
   - Accept Form fields: CALL, QSO_DATE, TIME_ON, BAND, FREQ, MODE, RST_SENT, RST_RCVD.
   - Accept optional query param: force (bool, default False).
   - Build QSO dict using build_qso_dict() with form data as dict and operator=callsign.
   - If not force: call find_duplicate(). If duplicate found, return HTTP 200 (NOT 409 — HTMX 2.x won't swap on 4xx) with templates/log/qso_result.html rendering the duplicate warning with "Save Anyway" button.
   - If no duplicate or force=true: insert QSO and return HTTP 200 with templates/log/qso_result.html showing success message with QSO details.
   - CRITICAL: Always return HTTP 200 for HTMX partials. Use template content to distinguish success vs warning.

Update app/main.py:
- Import: `from app.qso.ui_router import ui_router as qso_ui_router`
- Add: `app.include_router(qso_ui_router)` after the QSO API router include.
- The exception handler should already handle `/log/` prefix redirect from 03-01.
