Create the ADIF export endpoint that streams the operator's logbook as a valid .adi file. Uses StreamingResponse with a generator, filters by operator and excludes soft-deleted QSOs. Includes a _qso_to_adif_dict() helper that correctly maps QSO documents back to ADIF field dicts, skipping internal fields.

Purpose: Operators can download their logbook for backup or use in other logging software. Lossless export of APP_ and USERDEF fields ensures data portability.
Output: GET /api/adif/export returns a streaming .adi file download. UI link at /log/export.

<task type="auto">
  <name>Task 1: Create ADIF export endpoint with streaming response</name>
  <files>app/adif/router.py</files>
  <action>
Add to `app/adif/router.py`:

1. Create `_qso_to_adif_dict(qso: QSO) -> dict` helper function:
   - Start with empty dict `d = {}`
   - Add declared ADIF fields: `d["CALL"] = qso.CALL` (if not None), same for BAND, MODE
   - Iterate `qso.model_extra or {}`: for each key/value, skip if key is in `INTERNAL_FIELDS = {"qso_date_utc"}`. Skip if value is None. Otherwise `d[key] = str(val)`.
   - Do NOT include `operator_callsign`, `is_deleted`, `id`, `_id` — these are Beanie document fields, not ADIF fields. They are declared fields on the model, not in model_extra, so they won't appear in the iteration. But add a safety check: define `SKIP_FIELDS = {"qso_date_utc", "_operator", "_deleted", "_id", "id", "revision_id"}` and skip any key in that set during model_extra iteration.
   - Return `d`

2. Create `GET /export` endpoint on the existing router:
   - Depends on `get_current_operator_callsign`
   - Query: `qsos = await QSO.find({"_operator": operator, "_deleted": False}).to_list()`
   - Build ADIF header string: `<ADIF_VER:5>3.1.4\n<PROGRAMID:5>ollog\n<EOH>\n\n`
   - Use `StreamingResponse` with an async generator:
     - First yield the header
     - For each QSO, yield `serialize_adi([_qso_to_adif_dict(qso)])`
   - Set `media_type="text/plain"` and `Content-Disposition: attachment; filename="{operator}_logbook.adi"`

Import `StreamingResponse` from `fastapi.responses` and `serialize_adi` from `app.adif.serializer`.
  </action>

  </verify>
