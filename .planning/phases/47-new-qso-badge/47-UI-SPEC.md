## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon pulse dot dimensions (w-1.5 h-1.5 = 6px; nearest even token) |
| sm | 8px | Badge internal gap between icon and text |
| md | 16px | Default element spacing, filter form field gaps |
| lg | 24px | Section padding (card-body, space-y-6 column) |

| 2xl | 48px | Major section breaks |

Exceptions:
- Badge pill padding: `px-2.5 py-1` (10px / 4px) — matches existing LIVE indicator exactly. Source: `log.html` lines 18–23.

**Inherited values note:** The fractional Tailwind values `px-2.5` (10px), `gap-1.5` (6px), and `ml-1.5` (6px) are copied verbatim from the existing LIVE indicator component in production HTML (`log.html` lines 18–23). They are not new spacing choices introduced by Phase 47. This phase inherits them to maintain visual parity with the existing pill element.

## Component Inventory

### New Component: `#new-qso-badge`

**Type:** Inline pill element — DOM sibling of `#log-table`, not a child.

**HTML structure:**
```html
<div id="new-qso-badge"
     class="hidden items-center gap-1.5 px-2.5 py-1 rounded-full text-xs font-semibold
            bg-indigo-100 dark:bg-indigo-900/40 text-indigo-700 dark:text-indigo-300
            cursor-pointer select-none mb-2">
  <svg class="w-3.5 h-3.5" ...><!-- upward chevron or arrow icon --></svg>
  <span id="new-qso-badge-text">1 new QSO</span>
  <span class="ml-1 opacity-70">&times;</span>
</div>
```

**Placement:** Immediately before `<div id="log-table" ...>` inside the `max-w-7xl mx-auto space-y-6` wrapper. Must be a DOM sibling of `#log-table` — never a child. HTMX SSE swaps replace `#log-table` innerHTML and would destroy a badge nested inside.

Source: CONTEXT.md D-09; STATE.md §v2.4 Architecture Decisions "Badge placement".

**States:**
| State | Class | Condition |
|-------|-------|-----------|
| Hidden | `hidden` (no `flex`) | Default; page 1 with no filters; after dismiss |
| Visible | `hidden` removed, `flex` added | `new_qso` SSE event + `auto-refresh-ok` absent |

**Sizing reference:** Exactly matches `#live-indicator` pill sizing: `px-2.5 py-1 rounded-full text-xs font-semibold gap-1.5`. Source: `log.html` lines 18–23.

## Interaction Contract

### Badge Counter Logic

Variable: `newQsoCount` (integer, initialized to 0). Lives inside the existing IIFE in `log.html` alongside `eventsFlowing`, `audioCtx`, `userInteracted`.

```
On htmx:sseMessage where target.id === 'log-table' AND event type === 'new_qso':
  IF document.getElementById('auto-refresh-ok') is absent:
    newQsoCount++
    updateBadge()   // sets text, shows badge
```

### Badge Text Rule

```
newQsoCount === 1  →  "1 new QSO"
newQsoCount > 1   →  "{N} new QSOs"
```

Plural handled in JS: `newQsoCount === 1 ? '1 new QSO' : newQsoCount + ' new QSOs'`

Source: CONTEXT.md D-03; ROADMAP.md Phase 47 success criteria SC-2.

### Click-to-Dismiss

On `click` event on `#new-qso-badge`:
- `newQsoCount = 0`
- `dismissBadge()` — adds `hidden`, removes `flex`
- No page navigation, no scroll, no server request.

Source: CONTEXT.md D-07; REQUIREMENTS.md LIVE-04.

### Auto-Dismiss on htmx:afterSettle

New listener on `document.body` for `htmx:afterSettle`:
- If `document.getElementById('auto-refresh-ok')` is present (user navigated to page 1, default sort, no filters):
  - `newQsoCount = 0`
  - `dismissBadge()`

Source: CONTEXT.md D-08; STATE.md §v2.4 Architecture Decisions "Badge htmx:afterSettle re-sync".

### Edit Mode Behavior

Badge increments even when a row edit form is open on page 2+ (auto-refresh is already blocked by `if (document.querySelector('#log-table input')) return;` — badge operates independently of that guard). Source: CONTEXT.md D-06.

### Animation

Simple show/hide with `hidden` / `flex` class toggle. No CSS transition on the badge itself — keep consistent with LIVE indicator which also uses plain class swap. Executor MAY add a brief `transition-opacity` fade (e.g. `transition-opacity duration-150`) if it does not introduce new CSS classes not present in `output.css`.

Source: CONTEXT.md "Claude's Discretion" — animation is optional, keep simple.

### Icon

Upward chevron or arrow SVG (Heroicons, inline, `w-3.5 h-3.5`, `stroke-width: 2`, `stroke="currentColor"`). Either `chevron-up` or `arrow-up` is acceptable. Executor chooses based on visual balance with the LIVE indicator dot.

Source: CONTEXT.md "Claude's Discretion"; CONTEXT.md D-03 specifies `▲` symbol as reference.
