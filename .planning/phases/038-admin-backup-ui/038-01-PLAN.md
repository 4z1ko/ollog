Add the `backup_page` route handler to `app/admin/ui_router.py`. Insert it in the `# Backup download` section, **above** the existing `backup_download` endpoint (before line 229). The route serves the backup page template, protected by `require_admin_cookie`.

Add this handler:

```python
@ui_router.get("/backup", response_class=HTMLResponse)
async def backup_page(
    request: Request,
    _user: User = Depends(require_admin_cookie),
):
    """Render the admin backup page."""
    return templates.TemplateResponse(
        request,
        "admin/backup.html",
        {},
    )
```

Place it directly below the `# Backup download` section comment and above the existing `@ui_router.get("/backup/download")` handler.

CRITICAL: Use `Depends(require_admin_cookie)` — NOT `require_admin` (which expects a Bearer header, not a cookie). The existing imports already include `Depends`, `Request`, `HTMLResponse`, `require_admin_cookie`, and `User` — no new imports needed.

Note on route ordering: FastAPI matches routes in registration order. `/backup` must be registered before `/backup/download` so that `/backup` is not incorrectly matched as a path parameter. Since both are exact string matches (no path params), order doesn't strictly matter here, but placing the page route above the download route keeps the code logically organized (page before action).
