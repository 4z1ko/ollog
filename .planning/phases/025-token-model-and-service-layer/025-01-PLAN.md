<task type="auto">
  <name>Task 2: Token service helpers and test suite</name>
  <files>app/tokens/service.py, tests/test_tokens.py, .env</files>
  <action>
1. Create `app/tokens/service.py` with four public functions using only stdlib (`hmac`, `hashlib`, `secrets`, `re`):

   - `generate_api_token() -> tuple[str, str]`: `body = secrets.token_urlsafe(32)` (43 chars, 256-bit entropy), return `(f"ollog_{body}", body[:8])`. The tuple is `(full_token, token_prefix)`.

   - `hash_api_token(token: str) -> str`: `key = settings.api_token_secret.get_secret_value().encode()`, return `hmac.new(key, token.encode(), hashlib.sha256).hexdigest()`.

   - `verify_api_token(token: str, hashed: str) -> bool`: return `hmac.compare_digest(hash_api_token(token), hashed)`. This is constant-time because `hmac.compare_digest` prevents timing attacks.

   - `validate_token_name(name: str) -> str`: compile `_TOKEN_NAME_RE = re.compile(r'^[a-zA-Z0-9_-]{1,80}$')` at module level. If `not _TOKEN_NAME_RE.match(name)`, raise `ValueError("Token name must be 1-80 characters: alphanumeric, hyphens, underscores only")`. Return `name` on success.

   Import `settings` from `app.config`. Define `_PREFIX_LEN = 8` as a module constant.

2. Create `tests/test_tokens.py` with two sections:

   **Section 1 — Static tests (no MongoDB required):**
   - `test_api_token_collection_name`: assert `ApiToken.Settings.name == "api_tokens"`
   - `test_api_token_has_compound_index`: assert single index exists, check it contains both `token_prefix` and `user_id` keys (follow the pattern in `test_auth.py` `test_user_has_unique_username_index`)
   - `test_api_token_default_enabled`: assert `ApiToken.model_fields["enabled"].default is True`
   - `test_generate_token_format`: call `generate_api_token()`, assert full_token starts with `"ollog_"`, assert `len(full_token) == 49` (6 prefix + 43 body), assert `len(token_prefix) == 8`, assert `token_prefix == full_token[6:14]`
   - `test_generate_token_uniqueness`: call `generate_api_token()` twice, assert the two full tokens are different
   - `test_hash_api_token_returns_hex`: call `hash_api_token("ollog_test")`, assert result is a 64-char hex string (SHA-256 hexdigest length)
   - `test_verify_api_token_roundtrip`: generate a token, hash it, verify it — assert True
   - `test_verify_api_token_rejects_wrong`: generate a token, hash it, verify with a different token — assert False
   - `test_validate_token_name_accepts_valid`: test `"my-token"`, `"my_token"`, `"a"`, `"A" * 80` — all should return the name
   - `test_validate_token_name_rejects_invalid`: test `""`, `"A" * 81`, `"has spaces"`, `"has!bang"` — all should raise `ValueError`

   **Section 2 — Integration tests (MongoDB required):**
   - Create a `tokens_db` fixture following the `auth_db` pattern from `test_auth.py`: connect to `mongodb://localhost:27017/?directConnection=true`, create `ollog_test` database, call `init_beanie(database=db, document_models=[QSO, User, ApiToken])`, yield db, drop and close. Use `pytest.skip` if MongoDB is unavailable (same socket check as `conftest.py`).
   - `test_api_token_insert_and_find(tokens_db)`: create an `ApiToken` instance with dummy values (use a `PydanticObjectId()` for user_id, name="test-token", token_prefix="abcd1234", hashed_token="deadbeef"*8, enabled=True), insert it, find it back by token_prefix, assert fields match.

   **Important for env vars:** The test file needs `API_TOKEN_SECRET` set. Check how `test_auth.py` handles `SECRET_KEY`. The `.env` file already sets `SECRET_KEY`; adding `API_TOKEN_SECRET` to `.env` (from Task 1) will make it available to tests via pydantic-settings env_file loading. If that is insufficient, use `monkeypatch` or `os.environ` at the top of the test module to set `API_TOKEN_SECRET=test-secret` before importing `app.config`.

   Actually, since `app/config.py` instantiates `settings = Settings()` at module level, and `.env` is read by pydantic-settings, the `.env` value added in Task 1 will be picked up. But to be safe and explicit in tests, set `os.environ["API_TOKEN_SECRET"] = "test-token-secret"` before any app imports in the test file (same pattern as checking os.environ before imports seen in other test files).
  </action>
