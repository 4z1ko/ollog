# Plan 64-01: ACLog Bridge Manual Sync

**Execution status:** Complete 2026-06-12; see `64-01-SUMMARY.md` and `64-VERIFICATION.md`.

## Objective

Add manual ACLog synchronization from the operator Profile Settings page. A saved ACLog bridge row gets a Sync button that sends `<CMD><LIST><INCLUDEALL></CMD>` to that bridge, imports only QSOs that should be added to the logged-in operator's `<username>_qsos` collection, and renders a compact report in `#profile-result`.

## Boundaries

In scope:

- Saved-bridge-only Sync action in `templates/log/profile.html`.
- One synchronous HTMX route for a logged-in operator to sync one of their saved bridge IDs.
- Manual all-record ACLog TCP request using exactly `<CMD><LIST><INCLUDEALL></CMD>`.
- Parsing all returned full-record responses with existing ACLog parser helpers.
- Additive-only import through existing QSO ingest/duplicate/rowHash behavior.
- Compact report with "Missing QSOs imported", "Already present", "Errors", and first few rejected examples.
- Focused parser/client/profile tests and operator documentation update.

Out of scope:

- Background jobs, polling status, persisted sync history, or scheduled sync.
- Sync for unsaved `new-0` bridge rows.
- Updating, merging, or deleting existing local QSOs.
- Cross-operator sync or admin sync.
- Broad Profile Settings redesign or new UI libraries.

## Threat Model

### Assets

- Operator-owned QSO records in `<username>_qsos`.
- Operator ACLog bridge host/port configuration.
- ACLog TCP response payloads.
- Profile Settings session and HTMX result surface.

### Threats And Mitigations

- **T-64-01: Hung or malicious TCP peer keeps the request open.**  
  Mitigation: manual sync must use a fixed timeout and return a clear `ACLog sync failed` HTMX fragment.

- **T-64-02: Sync imports duplicates or overwrites local QSOs.**  
  Mitigation: sync is additive only; use existing `ingest_qso_record()` and rowHash/duplicate behavior; no update/delete code path is added.

- **T-64-03: Operator guesses another bridge ID.**  
  Mitigation: route resolves `bridge_id` only from authenticated `user.aclog_bridges` and writes only to `get_user_qso_collection(user)`.

- **T-64-04: Malformed ACLog payload causes route crash.**  
  Mitigation: parser/client failures should become rejected/error report entries or failure fragments, not unhandled template/route exceptions.

4. Press Sync again.

## Must Haves For Goal-Backward Verification

- Saved bridge rows expose a Sync action; the blank new row does not.
- Sync sends `<CMD><LIST><INCLUDEALL></CMD>` without `<VALUE>`.
- Sync imports accepted records into the logged-in user's collection only.
- Sync does not overwrite or delete existing QSOs.
- Report renders in `#profile-result` with the locked copy from UI-SPEC.
- Existing live ACLog bridge enrichment remains compatible with `<VALUE>5</VALUE>`.
