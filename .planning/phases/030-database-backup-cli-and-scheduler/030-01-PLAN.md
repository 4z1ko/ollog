Create the `app/backup/` directory with five files:

**`app/backup/__init__.py`** — empty file (package marker only).

**`app/backup/dump.py`** — export coroutine:
- `async def run_backup(settings) -> Path`
- Import `AsyncMongoClient` from `pymongo` directly — do NOT use `app.database.get_client()` (returns None in CLI context where lifespan has not run)
- Create client: `client = AsyncMongoClient(settings.mongodb_uri)`
- Construct output path: `Path(settings.backup_dir) / f"{datetime.utcnow().strftime('%Y%m%dT%H%M%SZ')}.gz"`
- Call `backup_path.parent.mkdir(parents=True, exist_ok=True)` before writing
- Use `gzip.open(backup_path, "wt", encoding="utf-8")` as context manager
- Inside the gzip file: iterate `sorted(await db.list_collection_names())`; for each collection call `await db[coll_name].find({}).to_list(length=None)`; for each document write one line: `dumps({"collection": coll_name, "doc": doc}, json_options=CANONICAL_JSON_OPTIONS) + "\n"` — one JSON object per document (NDJSON)
- Use `from bson.json_util import dumps, CANONICAL_JSON_OPTIONS` — NOT `RELAXED_JSON_OPTIONS` (relaxed loses Int64, Decimal128, Binary type fidelity)
- After the gzip context manager closes, call `client.close()`
- Return `backup_path`
- After return in the CLI path, if `settings.backup_s3_bucket` is not None, call `await upload_to_s3(backup_path, settings.backup_s3_bucket, f"{settings.backup_s3_prefix}{backup_path.name}")` (import from `app.backup.upload`)
- Add module-level logger: `logger = logging.getLogger(__name__)`

**`app/backup/upload.py`** — S3 upload:
- `async def upload_to_s3(local_path: Path, bucket: str, key: str) -> None`
- Import `aioboto3`
- `session = aioboto3.Session()`
- `async with session.client("s3") as s3:` then `with local_path.open("rb") as fp: await s3.upload_fileobj(fp, bucket, key)`
- Wrap the entire body in `try/except Exception as exc:` — on exception: `logger.error("S3 upload failed for %s: %s", local_path, exc)` and return (never re-raise; S3 failure must not propagate)
- Add module-level logger: `logger = logging.getLogger(__name__)`

**`app/backup/scheduler.py`** — APScheduler factory:
- `def make_scheduler(cron_expr: str, job_func) -> AsyncIOScheduler`
- `from apscheduler.schedulers.asyncio import AsyncIOScheduler`
- `from apscheduler.triggers.cron import CronTrigger`
- `scheduler = AsyncIOScheduler()`
- `scheduler.add_job(job_func, CronTrigger.from_crontab(cron_expr))`
- `return scheduler` — do NOT call `scheduler.start()` here; caller (lifespan) starts it
- Pin is in pyproject.toml (`apscheduler>=3.10,<4`); `AsyncIOScheduler` does not exist in v4 alpha

**`app/backup/__main__.py`** — CLI entry point:
- `async def main() -> None:`
  - `from app.config import settings`
  - `from app.backup.dump import run_backup`
  - `backup_path = await run_backup(settings)`
  - `print(str(backup_path))` — confirms path to stdout (BAK-01)
- `if __name__ == "__main__": import asyncio; asyncio.run(main())`
- The `asyncio.run()` creates its own event loop — this is correct for CLI; no lifespan is involved

**`app/main.py`** — wire scheduler into existing lifespan context manager.

In the `lifespan` function, BEFORE the `yield` statement (after the existing UDP block), add:

```python
# Start backup scheduler (conditional on BACKUP_SCHEDULE env var)
backup_task = None
backup_scheduler = None
if settings.backup_schedule:
    from app.backup.scheduler import make_scheduler
    from app.backup.dump import run_backup

    async def _backup_job():
        await run_backup(settings)

    backup_scheduler = make_scheduler(settings.backup_schedule, _backup_job)
    backup_scheduler.start()
    logger.info("Backup scheduler started (cron: %s)", settings.backup_schedule)
```

AFTER the `yield` statement (in the shutdown section, before `await close_db()`), add:

```python
if backup_task is not None:
    backup_task.cancel()
    try:
        await backup_task
    except asyncio.CancelledError:
        pass
if backup_scheduler is not None and backup_scheduler.running:
    backup_scheduler.shutdown(wait=False)
```

Note: The scheduler fires `_backup_job` via APScheduler's internal asyncio integration — `backup_task` here is `None` for the scheduler path (no separate `create_task` is needed; APScheduler manages job dispatch). The `backup_task` variable and its cancel block are present for correctness and forward compatibility but will be None unless a standalone async task is added later. Keep the pattern consistent with the research spec.
