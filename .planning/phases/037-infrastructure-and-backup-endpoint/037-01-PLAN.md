**app/admin/ui_router.py** (BACK-01, BACK-02, BACK-04):

Add a new route at the bottom of the file, after the existing `reset_password` endpoint, under a new section comment block.

1. Add imports at the top of the file (alongside existing imports):
   - `from fastapi.responses import FileResponse` (add to the existing `from fastapi.responses import HTMLResponse, RedirectResponse` line)

2. Add a section comment and the endpoint:
```
# ---------------------------------------------------------------------------
# Backup download
# ---------------------------------------------------------------------------

@ui_router.get("/backup/download")
async def backup_download(
    _user: User = Depends(require_admin_cookie),
):
    """Trigger a full MongoDB backup and return it as a .gz file download."""
    from app.backup.dump import run_backup
    from app.config import settings

    backup_path = await run_backup(settings)
    return FileResponse(
        path=backup_path,
        media_type="application/gzip",
        filename=f"ollog-backup-{backup_path.stem}.gz",
    )
```

Key details:
- Auth: `Depends(require_admin_cookie)` â€” NOT `require_admin` (Bearer). The browser sends a cookie, not an Authorization header. Using the wrong dependency causes silent 302 redirect to login. (BACK-02)
- Lazy imports for `run_backup` and `settings` inside the function body â€” matches the pattern used in `app/main.py` lifespan for backup imports, avoids circular import risk
- `FileResponse` automatically sets `Content-Disposition: attachment; filename="ollog-backup-2026-04-14-15-30-42.gz"` from the `filename=` parameter (BACK-04)
- `backup_path.stem` extracts the timestamp portion without `.gz` extension, then we re-add `.gz` with the `ollog-backup-` prefix â€” the filename matches the actual file on disk
- `run_backup` is awaited (it is async def), which internally calls `asyncio.to_thread(_write_backup, settings)` (BACK-03 satisfied by Task 1)

**What NOT to do:**
- Do NOT add `hx-*` attributes anywhere (this is a Python endpoint, not a template â€” but noted for Phase 38 awareness)
- Do NOT use `require_admin` (Bearer token dependency) â€” will cause silent 401 -> 302 redirect
- Do NOT call `_write_backup` directly â€” always go through `run_backup` which handles S3 upload if configured
- Do NOT add a second `datetime.now()` call for the filename â€” derive it from `backup_path.stem`
