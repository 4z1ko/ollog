<objective>
Add `.glass-card` to the Tailwind component layer with explicit `-webkit-backdrop-filter` support, apply it to both login templates, rebuild output.css, and verify Safari renders the blur.

Purpose: Both login pages currently use `backdrop-blur-sm` inline which compiles to a CSS variable reference. The Tailwind CLI standalone does NOT emit `-webkit-backdrop-filter`, so Safari pre-18.0 sees zero blur and Safari 18.x fails silently on CSS variable references inside `-webkit-backdrop-filter`. A single `.glass-card` class in `@layer components` with raw fixed-pixel values fixes both pages in one change.

Output: `static/css/input.css` (new `.glass-card` rule), `templates/admin/login.html` (card class replaced), `templates/log/login.html` (card class replaced), `static/css/output.css` (rebuilt with `-webkit-backdrop-filter`).
</objective>

  <name>Task 1: Add .glass-card component class to input.css</name>
  <files>static/css/input.css</files>
  <action>
    Append the following `.glass-card` block inside the existing `@layer components { ... }` block in `static/css/input.css`, after the last existing component class (before the closing brace of `@layer components`).

    ```css
    /* Glass card — Apple glassmorphism for login pages */
    /* Safari fix: -webkit-backdrop-filter requires fixed pixel values, NOT CSS variable references */
    /* Using raw CSS (not @apply) for backdrop-filter to avoid Tailwind CSS variable indirection */
    /* Reference: github.com/tailwindlabs/tailwindcss/issues/13844 */
    /* Reference: github.com/mdn/browser-compat-data/issues/25914 */
    .glass-card {
      @apply bg-white/10 rounded-2xl p-8 shadow-2xl;
      border: 1px solid rgba(255, 255, 255, 0.12);
      -webkit-backdrop-filter: blur(12px);
      backdrop-filter: blur(12px);
    }
    ```

    Critical constraints:
    - DO NOT use `@apply backdrop-blur-md` — Tailwind's backdrop-blur utilities generate `backdrop-filter: var(--tw-backdrop-blur) ...` which does not add a `-webkit-backdrop-filter` property. Safari pre-18.0 will get no blur.
    - DO NOT write `-webkit-backdrop-filter: var(--anything)` — Safari 18.x does not reliably resolve CSS variables inside `-webkit-backdrop-filter`. Only literal pixel values work.
    - The `blur(12px)` value corresponds to Tailwind's `backdrop-blur-md` as required by LOGN-01/LOGN-02.
    - `@apply bg-white/10 rounded-2xl p-8 shadow-2xl` is fine — these utilities generate standard properties with no webkit issues. `bg-white/10` (10% white) is used instead of the current `bg-white/5` (5%) for a more legible frosted appearance over the dark gradients.
    - The border is written as raw CSS to keep all glass-specific properties in one place; `border: 1px solid rgba(255,255,255,0.12)` is equivalent to Tailwind's `border border-white/10` (which is `rgba(255,255,255,0.1)`) — the raw value is slightly stronger at 0.12 for visibility.
  </action>
  <verify>
    Run: `grep -A 5 'glass-card' /Users/royco/ollog/static/css/input.css`

    Expected output must show both `-webkit-backdrop-filter: blur(12px)` and `backdrop-filter: blur(12px)` with literal pixel values, not variable references.
  </verify>
  <done>

    - `@apply bg-white/10 rounded-2xl p-8 shadow-2xl`
