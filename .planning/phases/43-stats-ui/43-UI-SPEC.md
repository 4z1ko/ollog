## Spacing Scale

Declared values (multiples of 4, Tailwind scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card body padding |
| lg | 24px | Section padding, card body (`px-6 py-5`) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Nav items: `py-2 px-3` (8px/12px) — matches established sidebar pattern.
- Card headers: `px-6 py-4` — matches established `card-header` class.
- Chart wrapper height: 256px (`h-64`) — fixed canvas height required by Chart.js `maintainAspectRatio: false` constraint (from STATE.md architecture decisions).

## Color

Source: `tailwind.config.js` color tokens, `input.css` component classes.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `#f2f2f7` (`canvas-light`) | `#0f0f0f` (`canvas-dark`) | Page background (`bg-canvas-light dark:bg-canvas-dark`) |
| Secondary (30%) | `#ffffff` (`surface-light`) | `#1c1c1e` (`surface-dark`) | Cards, summary row, chart containers |
| Accent (10%) | `#4f46e5` (indigo-600) | `#4f46e5` (indigo-600) | Primary button, focus rings, logo, user avatar |
| Destructive | `#e11d48` (rose-600) | `#e11d48` (rose-600) | Destructive actions only (none in this phase) |

Accent reserved for: primary CTA buttons, focus rings, sidebar active state, logo mark, user avatar. No accent usage on chart slices — chart colors are a separate palette (see Chart Color Palettes section below).

### Chart Color Palettes

Two palettes keyed to `PALETTES.dark` and `PALETTES.light`. Selected at chart-init via `document.documentElement.classList.contains('dark')`.

Dark palette (8 colors, muted/pastel):
`['#818cf8', '#34d399', '#fbbf24', '#60a5fa', '#f472b6', '#a78bfa', '#2dd4bf', '#fb923c']`
(indigo-400, emerald-400, amber-400, blue-400, pink-400, violet-400, teal-400, orange-400)

Light palette (8 colors, mid-saturation):
`['#4f46e5', '#059669', '#d97706', '#2563eb', '#db2777', '#7c3aed', '#0d9488', '#ea580c']`
(indigo-600, emerald-600, amber-600, blue-600, pink-600, violet-600, teal-600, orange-600)

Background fill per slice: same as border color at opacity 0.75 in dark mode, 0.85 in light mode. Chart.js `borderWidth: 2`, `borderColor: transparent` (no border lines between slices).

Chart text color: `'#9ca3af'` (gray-400) for labels in dark mode, `'#6b7280'` (gray-500) in light mode.

## Layout Contract

Source: `43-CONTEXT.md` §Decisions (D-01 and Claude's Discretion).

### Page Width
`max-w-5xl mx-auto` — wider than profile page (`max-w-3xl`) to accommodate three charts comfortably.

### Summary Row (above charts)
Single compact row, `max-w-5xl mx-auto`. Contains: "Total QSOs: N" in a `.card` with `.card-header` + `.card-body`. Renders using existing `card`, `card-header`, `card-title` component classes.

### Chart Grid
Two-column responsive grid for Band + Mode charts (top row):
```
<div class="grid grid-cols-1 md:grid-cols-2 gap-6">
  <!-- By Band chart card -->
  <!-- By Mode chart card -->
</div>
```
DXCC Entity chart spans full width below:
```
<div class="mt-6">
  <!-- By DXCC Entity chart card — full width -->
</div>
```
On mobile (`< md` breakpoint): all three cards stack vertically (single-column).

### DXCC Entity Scalar
Rendered as card subtitle inline with the chart title: `"By DXCC Entity · {N} entities"`. Not in a separate card. Example: `"By DXCC Entity · 42 entities"`. This keeps the chart and its scalar co-located as decided in CONTEXT.md.

### Canvas Sizing
Each `<canvas>` wrapped in `<div class="relative h-64 w-full">`. Chart.js option `maintainAspectRatio: false`. Required to prevent zero-width collapse in Tailwind flex/grid (from STATE.md architecture decisions).

## Dark Mode Re-init Contract

Source: `43-CONTEXT.md` §Decisions (dark mode hook), `base_app.html` `toggleTheme()`.

### Extension point in `base_app.html`
Append one line to the end of `toggleTheme()` function body:
```javascript
window.dispatchEvent(new CustomEvent('themechange'));
```
This does not change any existing `toggleTheme()` behavior — it adds a broadcast-only extension point.

### Listener in `stats.html` extra_scripts block
```javascript
window.addEventListener('themechange', reinitCharts);
```
`reinitCharts` destroys and recreates all three Chart instances with the updated palette.

### Stale canvas guard
Before every `new Chart(...)` call:
```javascript
Chart.getChart(canvas)?.destroy();
```
Required on bfcache restore and theme re-init (from STATE.md architecture decisions).

## Chart.js Delivery

Source: STATE.md §v2.3 Architecture Decisions.

- CDN URL: `https://cdn.jsdelivr.net/npm/chart.js@4.5.1/dist/chart.umd.min.js`
- Load location: `{% block extra_scripts %}` in `stats.html` only — never in `base.html`
- Bundle format: UMD (not ESM — ESM-only silently fails with "Chart is not defined")
- Chart types: three `'pie'` charts

Inline data injection (must use `| tojson`, never `| safe`):
```javascript
const bandData   = {{ band_counts   | tojson }};
const modeData   = {{ mode_counts   | tojson }};
const entityData = {{ entity_counts | tojson }};
```

## Empty State Contract

Condition: `total_qsos == 0`

Render: a single `.card` containing centered text with empty-state copy. No charts rendered. No canvas elements present in DOM.

```html
<div class="card">
  <div class="p-6 text-center text-gray-500 dark:text-gray-400">
    No data yet. Start logging QSOs to see your statistics here.
  </div>
</div>
```

No Chart.js script needed when in empty state — the `{% block extra_scripts %}` block should guard on `total_qsos > 0` or simply check for canvas presence before initializing.

## Accessibility Contract

- Each `<canvas>` element: `role="img"` with `aria-label="QSO count by {dimension} — pie chart"`
- Nav link: no `aria-label` needed (text label "Stats" is descriptive)
- Chart.js legend: enabled (`display: true`) with `position: 'bottom'` — provides text alternatives to color-coded slices
- Color contrast: chart label text uses gray-400 (dark) / gray-500 (light) against surface backgrounds — meets WCAG AA for informational text at 14px
