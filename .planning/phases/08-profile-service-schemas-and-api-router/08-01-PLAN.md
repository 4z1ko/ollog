Create app/profile/schemas.py with two Pydantic v2 models:

**ProfileUpdateRequest** — used for PATCH /api/profile body:

```python
import re
from typing import Optional

from pydantic import BaseModel, EmailStr, field_validator

MY_GRIDSQUARE_RE = re.compile(r"^[A-Ra-r]{2}[0-9]{2}([A-Xa-x]{2})?$")


class ProfileUpdateRequest(BaseModel):
    station_callsign: Optional[str] = None
    name: Optional[str] = None
    email: Optional[EmailStr] = None
    qth: Optional[str] = None
    state: Optional[str] = None
    country: Optional[str] = None
    my_gridsquare: Optional[str] = None
    my_rig: Optional[str] = None
    my_antenna: Optional[str] = None
    tx_pwr: Optional[float] = None

    @field_validator("my_gridsquare")
    @classmethod
    def validate_gridsquare(cls, v: str | None) -> str | None:
        if v is not None and not MY_GRIDSQUARE_RE.match(v):
            raise ValueError(f"Invalid Maidenhead grid format: {v!r}")
        return v.upper() if v else v

    @field_validator("station_callsign")
    @classmethod
    def normalize_station_callsign(cls, v: str | None) -> str | None:
        if v is not None and v.strip() == "":
            return None
        return v
```

Key design decisions:
- All fields Optional with None default — PATCH uses `model_dump(exclude_unset=True)` so absent fields are not sent as updates.
- `my_gridsquare` validator accepts 4-char ("FN31") and 6-char ("FN31pr") Maidenhead locators only. The regex `[A-Ra-r]{2}[0-9]{2}([A-Xa-x]{2})?$` enforces valid character classes. Auto-uppercases on validation.
- `station_callsign` validator normalizes empty string to None — prevents LoTW/POTA upload failures from blank STATION_CALLSIGN.
- Do NOT include `latitude`, `longitude`, or `callsign` — these are read-only (derived or identity fields).

**ProfileResponse** — used for GET and PATCH response:

```python
class ProfileResponse(BaseModel):

    name: Optional[str] = None
    email: Optional[EmailStr] = None
    qth: Optional[str] = None
    state: Optional[str] = None

    my_gridsquare: Optional[str] = None
    latitude: Optional[float] = None
    longitude: Optional[float] = None
    my_rig: Optional[str] = None
    my_antenna: Optional[str] = None
    tx_pwr: Optional[float] = None
```

Key design decisions:
- `callsign` is str (not Optional) — always present from User.callsign.
- `latitude` and `longitude` are read-only computed fields — included in response but not in update request.
- Field names match User model fields exactly for `model_validate(user.model_dump())` compatibility.
