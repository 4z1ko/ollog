<objective>
Add favicon support across all ollog full-page web surfaces by serving `favicon/favicon.ico` through existing static paths, adding one shared app-page favicon link in `templates/base.html`, configuring MkDocs `/guide` favicon behavior, rebuilding committed guide output, and verifying no HTMX partials or unrelated behavior are changed.
</objective>

<threat_model>
## Threat Model

**Assets:** static file serving, shared HTML head, MkDocs generated `site/` output.

**Trust boundaries:** browser requests to `/static/...` and `/guide/...` cross from public HTTP clients into files committed in the repository.

**Threats considered:**
- **Path exposure:** avoid adding a new broad `StaticFiles` mount for the source `favicon/` folder; use existing `/static` and MkDocs asset output only.
- **Template injection/regression:** add a fixed literal favicon URL only; do not introduce user-controlled template values or inline scripts.
- **Generated-site drift:** rebuild `/guide` from `mkdocs.yml` and committed docs sources so generated output remains reproducible.

**Residual risk:** low. This phase serves static favicon assets and fixed metadata only.
</threat_model>
