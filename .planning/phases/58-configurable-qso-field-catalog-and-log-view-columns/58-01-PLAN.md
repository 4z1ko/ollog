# Phase 58 Plan 01: Configurable QSO Field Catalog and Log View Columns

## Objective

Replace the Log View's hard-coded eight-column selector and table cells with a shared known QSO/ADIF field catalog so operators can select all supported display fields from the existing gear menu, while preserving current defaults, localStorage persistence, HTMX partial swaps, SSE refresh behavior, sorting for existing sortable fields, and the always-visible Actions column.

## Context

Read before execution:

- `.planning/phases/58-configurable-qso-field-catalog-and-log-view-columns/58-CONTEXT.md`
- `.planning/phases/58-configurable-qso-field-catalog-and-log-view-columns/58-RESEARCH.md`
- `.planning/phases/58-configurable-qso-field-catalog-and-log-view-columns/58-UI-SPEC.md`
- `.planning/phases/58-configurable-qso-field-catalog-and-log-view-columns/58-VALIDATION.md`

## Must Haves

- The catalog is curated from known ADIF/common QSO fields only; it does not discover fields from current MongoDB documents.
- Fresh browsers show exactly the current defaults: `date`, `call`, `band`, `mode`, `freq`, `rst`.
- Additional selected fields append after defaults in catalog order.
- The Actions column remains visible and outside the configurable catalog loop.
- Sort controls remain limited to Date / Time, entry timestamp, Callsign, Band, and Mode.
- Missing values render as blank cells.
- The gear menu remains one bounded scrollable checklist using existing visual tokens.
- `ollog.log.columns` remains the browser persistence key.

## Threat Model

| Threat | Risk | Mitigation |
|--------|------|------------|
| T-58-01 unsafe field exposure | Internal fields such as `_deleted`, `rowHash`, raw IDs, or credential-like values become selectable. | Catalog is curated allowlist only; tests assert unsafe keys are absent. |
| T-58-02 row action loss | Generated table loops accidentally hide or move Edit/Delete controls. | Actions column remains outside configurable field loop; tests assert action controls remain in rendered table. |
| T-58-03 stale localStorage breakage | Old or invalid persisted keys hide all useful columns or desync checkboxes. | JS normalizes persisted keys against server-emitted `CONFIGURABLE_COLUMNS` and falls back to defaults when no valid keys remain. |
| T-58-04 sort scope creep | New selected fields bypass service sort allowlist or create inefficient arbitrary Mongo sorts. | Do not expand `_ALLOWED_SORT_FIELDS`; non-sortable catalog entries render plain headers. |
| T-58-05 XSS via extra ADIF values | Arbitrary ADIF extras render unsafe HTML. | Keep values as strings rendered through Jinja escaping; do not mark field values safe. |
