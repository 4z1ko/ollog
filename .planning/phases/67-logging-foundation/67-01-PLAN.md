# Phase 67 Plan: Logging Foundation

## Goal

Add the backend foundation for internal application logging: MongoDB models, threshold settings, reusable async logger, sensitive-value masking, retention, indexes, and live broadcast plumbing.

## Scope

- Create `app/internal_logs/` package.
- Add `ApplicationLog` and `ApplicationLogSettings` Beanie documents.
- Register the new documents during database initialization.
- Implement severity ordering: `Trace`, `Debug`, `Info`, `Warn`, `Error`, `Fatal`.
- Default minimum level: `Info`.
- Default retention: 30 days.
- Store logs in MongoDB only when at or above threshold.
- Add structured context fields and metadata sanitization.
- Add broadcast manager for future admin SSE log viewer.
- Add focused unit tests that do not require a live MongoDB where possible.

## Implementation Tasks

1. Add log models and constants.
2. Add service functions for settings lookup/update, level comparison, masking, record construction, and safe insert.
3. Add TTL-style `expires_at` field and index.
4. Register models in `app/database.py`.
5. Add tests for level thresholding, masking, record shape, and retention timestamp.

## Safety Notes

- Logger must never raise into ingestion flows.
- Passwords, API keys, tokens, secrets, and MongoDB URI credentials must be masked before storage.
- Existing Python process logging remains unchanged.
