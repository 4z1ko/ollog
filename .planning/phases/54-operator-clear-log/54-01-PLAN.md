```python
async def clear_operator_log(operator: str) -> int:
    """Permanently delete all active (non-soft-deleted) QSOs for an operator.

    Returns the count of deleted documents. Permanent delete (not soft-delete)
    per CLR-03 requirements — sets do not toggle `_deleted`; documents are
    removed entirely from MongoDB.

    Args:
        operator: The operator callsign (from JWT cookie / current user).

    Returns:
        Number of QSO documents removed. Zero is a valid, non-error return value
        (operator may legitimately have an empty log).
    """
    result = await QSO.find(
        {"_operator": operator, "_deleted": False}
    ).delete_many()
    return result.deleted_count if result is not None else 0
```

The filter `{"_operator": operator, "_deleted": False}` MUST include both keys exactly as
written — operator isolation depends on `_operator`, and excluding already-soft-deleted
records prevents the deleted_count from drifting away from the visible count shown in the
modal (per RESEARCH.md Pitfall 3 + Assumption A1).

The `result is not None` guard handles the documented `DeleteMany | DeleteResult | None`
return shape from Beanie (RESEARCH.md Pitfall 2).

Do NOT add a try/except around the delete — Beanie/PyMongo errors should propagate so
FastAPI returns a 500 (the route handler does not need to swallow them; if MongoDB is
unreachable, that is a startup-level failure, not a per-request concern).

Do NOT modify the imports block — `QSO` is already imported at line 9 of service.py.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-01 | Tampering | `clear_operator_log(operator)` filter in app/qso/service.py | mitigate | Filter dict literal `{"_operator": operator, "_deleted": False}`; `operator` parameter sourced exclusively from authenticated `user.callsign` (verified in Plan 02 route); never from request body |
