## Layout

The Danger Zone card appears as the **last card** in `templates/log/profile.html`, below the API Tokens section. It follows the same `max-w-3xl mx-auto space-y-6` container that wraps all profile cards.

The confirmation modal uses the existing `.modal-backdrop` + `.modal-box` pattern established in `templates/admin/restore/password_modal.html`:
- Backdrop: `fixed inset-0 bg-black/50 z-40` with `backdrop-filter: blur(4px)` + `-webkit-backdrop-filter: blur(4px)`
- Box: `fixed top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2 z-50 bg-surface rounded-xl shadow-2xl p-6 w-full max-w-md`

The modal is injected into the page via HTMX. A dedicated `<div id="clear-log-modal"></div>` target lives outside the profile card container so HTMX swaps do not disturb surrounding cards.

---

## Focal Points

**Danger Zone card:** The `.btn-danger` rose-600 button ("Clear my log") is the sole focal point of this card. All other content — the card title and description text — exists to contextualise and guard that button. No competing visual weight should appear adjacent to it.

**Confirmation modal:** The modal title ("Clear My Log") paired with the bolded QSO count (`<strong>{N} QSO(s)</strong>`) inside `.modal-body` is the focal point. Together they communicate the irreversibility and exact scope of the action before the operator commits.

## Interaction Flow

### Trigger
1. Operator clicks "Clear my log" button (`btn-danger`) in the Danger Zone card.
2. HTMX fires `hx-get="/log/profile/clear/modal"` targeting `#clear-log-modal` with `hx-swap="innerHTML"`.
3. Server returns the modal HTML fragment (HTTP 200). The `#clear-log-modal` div fills with the backdrop + box.

### Modal — pending state
- Modal displays: "This will permanently delete **{N} QSO(s)** from your log. This cannot be undone."
- QSO count is bolded using `<strong>` inside `.modal-body`.
- Password field (`.form-control`) with label "Your password", `autocomplete="current-password"`, `type="password"`, required.
- Inline error target `<div id="clear-log-error"></div>` above the password field.
- Action row: "Delete {N} QSOs" (`btn-danger`) + "Keep my log" (`btn-secondary`).

### Submit — correct password
1. HTMX posts to `POST /log/profile/clear` targeting `#clear-log-modal` with `hx-swap="outerHTML"`.
2. Server deletes all operator QSOs, returns success fragment (HTTP 200).
3. Success fragment replaces the entire modal element: `<div id="clear-log-modal">` containing an `.alert-success` showing the count.
4. Backdrop is gone — no DOM removal JavaScript needed.

### Submit — wrong password
1. Server returns the same modal HTML with `#clear-log-error` populated: `<div class="alert alert-error" role="alert">Incorrect password — no QSOs were deleted.</div>`.
2. HTMX swaps `outerHTML` of `#clear-log-modal` — modal stays visible with error shown above the password field.

### Cancel
- "Keep my log" button: plain `type="button"` with `onclick="document.getElementById('clear-log-modal').innerHTML = ''"` — removes modal from DOM without a server round-trip.
- No page reload. Profile cards remain fully intact.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Section card title | "Danger Zone" |
| Section description | "Permanently delete all QSOs from your log. This cannot be undone." |
| Primary CTA (Danger Zone button) | "Clear my log" |

| Modal body (N > 0) | "This will permanently delete **{N} QSO(s)** from your log. This cannot be undone." |

| Password field placeholder | "Enter your password to confirm" |
| Modal submit button (N > 0) | "Delete {N} QSOs" |
| Modal submit button (N = 0) | "Confirm (0 QSOs)" |

| Wrong password error (inside modal) | "Incorrect password — no QSOs were deleted." |
| Success message (after deletion, N > 0) | "Done. {N} QSO(s) deleted from your log." |

## Accessibility

- "Clear my log" button must carry `aria-label="Clear my log — opens confirmation modal"`.
- Modal container sets `role="dialog"` and `aria-modal="true"` and `aria-labelledby="clear-log-modal-title"`.
- `<h3 id="clear-log-modal-title" class="modal-title">Clear My Log</h3>` is the label target.
- Password input: `id="clear-log-password"` with `<label for="clear-log-password">`.
- Error alert: `role="alert"` so screen readers announce it on injection.
- Cancel button: `type="button"` (not submit) to prevent accidental form submission.
