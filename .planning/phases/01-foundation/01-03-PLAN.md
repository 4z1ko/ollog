Create app/utils.py with from_mongo_dt() function:
```python
from datetime import datetime, timezone

def from_mongo_dt(dt: datetime | None) -> datetime | None:
    """Re-attach UTC tzinfo to a datetime read from MongoDB.
    PyMongo may return naive datetimes — always call this after reads."""
    if dt is None:
        return None
    if dt.tzinfo is None:
        return dt.replace(tzinfo=timezone.utc)
    return dt
```

- Import: pymongo, IndexModel from pymongo, Document from beanie, ConfigDict from pydantic, Field from pydantic, Optional from typing, datetime
- class QSO(Document):
  - model_config = ConfigDict(extra="allow", populate_by_name=True)
  - Declared fields (for indexing and type safety):
    - operator_callsign: str = Field(serialization_alias="_operator")
      NOTE: Use `serialization_alias="_operator"` so that when Beanie serializes to MongoDB, the document field is `_operator`. Combined with `populate_by_name=True` in ConfigDict, this allows creating QSO instances using either `operator_callsign="XX"` or `_operator="XX"`. The MongoDB document MUST have `_operator` as the field name — no fallback to "operator" allowed per locked decision.
    - CALL: str
    - BAND: str | None = None
    - MODE: str | None = None
    - qso_date_utc: datetime | None = None
    - is_deleted: bool = Field(default=False, serialization_alias="_deleted")
      Same pattern: Python field `is_deleted`, MongoDB field `_deleted`.
  - @classmethod
    async def find_active(cls, operator: str) -> list["QSO"]:
        """Find all non-deleted QSOs for an operator. Use this instead of raw find() to exclude soft-deleted records by default."""
        return await cls.find({"_operator": operator, "_deleted": False}).to_list()
  - class Settings:
    - name = "qsos"
    - indexes = [
        IndexModel(
            [("_operator", pymongo.ASCENDING), ("CALL", pymongo.ASCENDING), ("qso_date_utc", pymongo.ASCENDING), ("BAND", pymongo.ASCENDING), ("MODE", pymongo.ASCENDING)],
            unique=True,
            name="operator_qso_unique",
        ),
        IndexModel([("_operator", pymongo.ASCENDING)], name="operator_idx"),
        IndexModel([("_operator", pymongo.ASCENDING), ("_deleted", pymongo.ASCENDING)], name="operator_active_idx"),
      ]

IMPORTANT: The MongoDB documents MUST use `_operator` and `_deleted` as field names. This is achieved via `serialization_alias` + `populate_by_name=True`. If Beanie 2.x has issues with serialization_alias for Document fields, use Beanie's own `Field(alias="_operator")` instead — but test to confirm the alias actually appears in MongoDB documents. There is NO fallback to storing without the underscore prefix. The `_operator` field name in MongoDB is a locked decision.
