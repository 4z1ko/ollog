  <name>ADIF Parser and Serializer</name>

  <behavior>
    parse_adi(text: str) -> tuple[list[dict[str, str]], list[dict]]:
      - Parses ADIF tag-stream format: &lt;FIELDNAME:LENGTH&gt;VALUE or &lt;FIELDNAME:LENGTH:TYPE&gt;VALUE
      - Returns (records, errors) where records are dicts with UPPERCASE field names
      - Handles &lt;EOH&gt; (end of header) — skips all content before it
      - Handles missing &lt;EOH&gt; — if no EOH found, entire content is record data
      - Handles &lt;EOR&gt; (end of record) — finalizes current record
      - Field names normalized to UPPERCASE (ADIF spec: case-insensitive)
      - APP_ fields and USERDEF fields preserved verbatim (not dropped)
      - Single-record parse errors collected in errors list, do not abort file
      - Whitespace between tags is ignored (not part of field values)

    serialize_adi(records: list[dict[str, str]], header: str | None = None) -> str:
      - Produces valid ADIF tag-stream output
      - Field lengths use len(value.encode('utf-8')) — NOT len(value) (user locked decision)
      - Each record terminated with &lt;EOR&gt;
      - Optional header terminated with &lt;EOH&gt;
      - Field order: deterministic (sorted by field name for reproducibility)

    Test cases:
      - Simple ASCII: "&lt;CALL:4&gt;W1AW&lt;BAND:3&gt;20M&lt;EOR&gt;" -> [{"CALL": "W1AW", "BAND": "20M"}]
      - Non-ASCII: "&lt;NAME:6&gt;Andre\u0301" with proper UTF-8 byte length handling
      - APP_ fields: "&lt;APP_MYLOGGER_SCORE:3&gt;100" preserved in output
      - Missing EOH: file with no &lt;EOH&gt; tag still parses records
      - Case insensitive: "&lt;call:4&gt;W1AW" normalizes to CALL
      - Bad record: malformed length continues parsing remaining records
      - Round-trip: parse(serialize(parse(text))) == parse(text) — record dicts are identical after round-trip (field ordering in serialized output may differ from original, so we compare parsed dicts, not raw strings)
      - Multi-byte: name "Muller" with umlaut u (2 UTF-8 bytes) round-trips correctly
  </behavior>
  <implementation>
    Parser: State machine that scans for '&lt;', extracts tag content up to '&gt;', splits on ':' to get field name and byte length, reads exactly byte_length characters of value data. Track in_header state (True until EOH seen). Collect current_record fields until EOR. Per research Pattern 5.

    Serializer: For each record, iterate sorted field names, compute byte length as len(value.encode('utf-8')), emit &lt;FIELDNAME:LENGTH&gt;VALUE, end with &lt;EOR&gt; and newline.

    CRITICAL per locked decision: serializer MUST use len(value.encode('utf-8')), not len(value).
    CRITICAL per locked decision: parser MUST normalize field names to uppercase.
    CRITICAL per locked decision: APP_ and USERDEF fields MUST NOT be filtered out.
    CRITICAL per locked decision: parser errors on single record MUST NOT abort the file.
  </implementation>

Create app/adif/parser.py implementing parse_adi(text: str) -> tuple[list[dict[str, str]], list[dict]]:
- State machine: scan character by character for '&lt;' tag openings
- On finding '&lt;', find matching '&gt;', extract tag_content
- Split tag_content on ':' — parts[0] is field name (uppercase it), parts[1] is byte length, optional parts[2] is type indicator
- Special tags: EOH sets in_header=False, EOR appends current_record to records and resets
- For data tags: parse byte_length as int, read the next byte_length characters as the value
- If byte_length parse fails: append error dict with record_index and error message, continue
- Skip content while in_header=True (before EOH)
- Handle missing EOH: if no EOH encountered, treat all content as records (set in_header=False from start? No — check if file has any '&lt;EOH&gt;' first, or simply: if we reach EOR while still in_header, that means there was no EOH, so set in_header=False retroactively). Simplest approach: start with in_header=True, but when we encounter a data field (not EOH/EOR) while in_header and no EOH has been seen yet, check if EOH exists anywhere in the remaining text. If not, set in_header=False. OR: simpler — pre-scan for EOH existence. If no EOH in text, start with in_header=False.
- Return (records, errors)

NOTE on byte length vs character indexing: The ADIF length field L in &lt;F:L&gt; is a byte count. When reading from a Python str (already decoded), if the file was read as UTF-8, then character positions may not match byte positions for multi-byte chars. For parsing: read the raw bytes, or more practically — since we receive a str, we need to handle this carefully. The safest approach for the parser:
  1. Work with bytes internally: encode the text to UTF-8 bytes, scan for tags in bytes, extract byte_length bytes of value, decode back to str.
  OR
  2. Work with str but when extracting value of byte_length bytes: encode the remaining text to bytes, take byte_length bytes, decode to str, advance the str position by the number of characters consumed.

Approach 2 is cleaner. After finding a tag with byte_length:
  remaining_bytes = text[i:].encode('utf-8')
  value_bytes = remaining_bytes[:byte_length]
  value = value_bytes.decode('utf-8')
  i += len(value)  # advance by character count, not byte count

Create app/adif/serializer.py implementing serialize_adi(records: list[dict[str, str]], header: str | None = None) -> str:
- If header provided, emit header text + "\n&lt;EOH&gt;\n\n"
- For each record: iterate sorted(record.keys()), for each field emit "&lt;{FIELD}:{byte_len}&gt;{value}" where byte_len = len(value.encode('utf-8'))
- After all fields in record, emit "&lt;EOR&gt;\n\n"
- Return the full string
