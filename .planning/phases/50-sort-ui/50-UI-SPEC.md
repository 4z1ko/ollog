## Interaction Contract

This section is the primary deliverable for Phase 50. It specifies every new interactive element
added to `templates/log/log_table.html`.

### 1. Inactive Sort Indicator (UX-01)

Applied to: DATE text link, CALL, BAND, MODE, and the clock icon link — when that element is NOT
the active sort.

**SVG:** Heroicons `chevrons-up-down` outline, 20px viewBox, rendered at `w-3 h-3`.

```html
<svg class="w-3 h-3 opacity-30 dark:opacity-25" fill="none" viewBox="0 0 20 20" stroke-width="1.5" stroke="currentColor">
  <path stroke-linecap="round" stroke-linejoin="round" d="M8 9l4-4 4 4m0 6l-4 4-4-4" />
</svg>
```

- Light mode: `opacity-30` — faint but perceptible against light `#f2f2f7` canvas.
- Dark mode: `dark:opacity-25` — slightly more transparent to avoid over-prominence against dark surface.
- The icon uses `stroke="currentColor"` (outline style), NOT `fill="currentColor"` (solid).
  The distinction between outline (inactive) and solid fill (active) is the primary visual affordance for UX-01 vs UX-02.
- Position: appended inline after the column label text (or after the clock SVG), inside the same `<a>` element, following the existing `gap-1` text+icon pattern.

**Build gate:** After adding `dark:opacity-25` — run `npm run verify` and confirm `dark:opacity-25`
appears in `static/css/output.css`. This class is new to the project.

### 2. Active Sort Indicator (UX-02)

Applied to: the currently active sort column's `<a>` element only. Identical SVG blocks already
exist in `log_table.html` for DATE, CALL, and BAND — Phase 50 extends to MODE and the clock icon.

**Active descending (filled chevron-down):**
```html
<svg class="w-3 h-3" fill="currentColor" viewBox="0 0 20 20">

</svg>
```

**Active ascending (filled chevron-up):**
```html
<svg class="w-3 h-3" fill="currentColor" viewBox="0 0 20 20">
  <path fill-rule="evenodd" d="M14.77 12.79a.75.75 0 01-1.06-.02L10 8.832l-3.71 3.938a.75.75 0 11-1.08-1.04l4.25-4.5a.75.75 0 011.08 0l4.25 4.5a.75.75 0 01-.02 1.06z" clip-rule="evenodd"/>
</svg>
```

These SVG paths are copied verbatim from the existing DATE/CALL/BAND active chevrons in

### 3. MODE Column Sort Header (SORT-01)

**Before (Phase 49 state):**
```html
<th>Mode</th>
```

**After (Phase 50):**
```html
<th>
  <a class="inline-flex items-center gap-1 hover:text-indigo-400 transition-colors cursor-pointer"
     hx-get="/log/view?sort={% if sort == 'MODE' %}-MODE{% else %}MODE{% endif %}&call={{ filters.call or '' }}&band={{ filters.band or '' }}&mode={{ filters.mode or '' }}&date_from={{ filters.date_from or '' }}&date_to={{ filters.date_to or '' }}"
     hx-target="#log-table"
     hx-swap="innerHTML"
     hx-push-url="true">
    Mode
    {% if sort == '-MODE' %}
    <!-- active desc: solid chevron-down -->
    <svg class="w-3 h-3" fill="currentColor" viewBox="0 0 20 20"><path fill-rule="evenodd" d="M5.23 7.21a.75.75 0 011.06.02L10 11.168l3.71-3.938a.75.75 0 111.08 1.04l-4.25 4.5a.75.75 0 01-1.08 0l-4.25-4.5a.75.75 0 01.02-1.06z" clip-rule="evenodd"/></svg>
    {% elif sort == 'MODE' %}
    <!-- active asc: solid chevron-up -->
    <svg class="w-3 h-3" fill="currentColor" viewBox="0 0 20 20"><path fill-rule="evenodd" d="M14.77 12.79a.75.75 0 01-1.06-.02L10 8.832l-3.71 3.938a.75.75 0 11-1.08-1.04l4.25-4.5a.75.75 0 011.08 0l4.25 4.5a.75.75 0 01-.02 1.06z" clip-rule="evenodd"/></svg>
    {% else %}
    <!-- inactive: hollow chevrons-up-down -->
    <svg class="w-3 h-3 opacity-30 dark:opacity-25" fill="none" viewBox="0 0 20 20" stroke-width="1.5" stroke="currentColor"><path stroke-linecap="round" stroke-linejoin="round" d="M8 9l4-4 4 4m0 6l-4 4-4-4" /></svg>
    {% endif %}
  </a>
</th>
```

**Sort direction toggle rule (D-09):** `MODE` ascending-first. First click → `?sort=MODE` (asc,
A→Z). Second click → `?sort=-MODE` (desc, Z→A). This differs from CALL/BAND which are
descending-first. CALL and BAND are NOT changed.

**HTMX attributes:** Identical to existing CALL/BAND sort links — `hx-target="#log-table"`,
`hx-swap="innerHTML"`, `hx-push-url="true"`.

**Filter preservation:** All five filter params (`call`, `band`, `mode`, `date_from`, `date_to`)
appended verbatim — same pattern as DATE/CALL/BAND.

### 4. DATE Header Restructure (SORT-02)

The DATE `<th>` is restructured from a single `<a>` to a flex container holding two independent
`<a>` elements.

**Flex wrapper:**
```html
<th>
  <span class="inline-flex items-center gap-2">
    <!-- link A: date sort -->
    <a class="inline-flex items-center gap-1 hover:text-indigo-400 transition-colors cursor-pointer" ...>
      Date / Time UTC
      <!-- chevron: active or inactive per sort state -->
    </a>
    <!-- link B: clock icon (_created_at sort) -->
    <a class="inline-flex items-center gap-1 hover:text-indigo-400 transition-colors cursor-pointer" ...>
      <!-- clock SVG -->
      <!-- chevron: active or inactive per sort state -->
    </a>
  </span>
</th>
```

**Wrapper element:** `<span class="inline-flex items-center gap-2">` — `gap-2` (8px) separates the
two links to make their independence visually clear. The executor may use this span or rely on the
`<th>`'s own flex context if the `<th>` already has `inline-flex` — the semantic requirement is
8px gap between the two `<a>` elements.

**Link A — Date sort (unchanged toggle logic):**
```
sort={% if sort == '-qso_date_utc' %}qso_date_utc{% else %}-qso_date_utc{% endif %}
```
- When `sort == '-qso_date_utc'`: show solid chevron-down (desc active).
- When `sort == 'qso_date_utc'`: show solid chevron-up (asc active).
- Otherwise (`sort` is CALL, BAND, MODE, or `_created_at`): show hollow `chevrons-up-down` at
  `opacity-30 dark:opacity-25`.

**Link B — Clock icon (`_created_at` sort, D-03 to D-05):**

First click → `-_created_at` (newest-entered first, descending). Second click → `_created_at`
(oldest-entered first, ascending).

```
sort={% if sort == '-_created_at' %}_created_at{% else %}-_created_at{% endif %}
```

Clock SVG (Heroicons `clock` outline, 20px viewBox, rendered `w-4 h-4` — one size up from the
chevrons to give the icon visual weight as a standalone glyph):
```html
<svg class="w-4 h-4" fill="none" viewBox="0 0 24 24" stroke-width="1.5" stroke="currentColor">
  <path stroke-linecap="round" stroke-linejoin="round" d="M12 6v6h4.5m4.5 0a9 9 0 1 1-18 0 9 9 0 0 1 18 0Z" />
</svg>
```

Clock icon sort states:
- When `sort == '-_created_at'`: clock SVG + solid chevron-down (desc active).
- When `sort == '_created_at'`: clock SVG + solid chevron-up (asc active).
- Otherwise: clock SVG + hollow `chevrons-up-down` at `opacity-30 dark:opacity-25`.

The clock SVG itself always renders at full opacity regardless of sort state — it is the sort
direction chevron that carries the active/inactive visual signal.

**HTMX attributes for both links:** Identical to existing sort links — `hx-target="#log-table"`,
`hx-swap="innerHTML"`, `hx-push-url="true"`. All five filter params appended.

### 5. Inactive indicator on existing CALL and BAND headers

## Accessibility Notes

- All sort `<a>` elements already have visible text labels (DATE text, Callsign, Band) or an icon
  that conveys affordance through context. The clock icon has no explicit `aria-label` — this is
  acceptable for the current scope (CONTEXT.md defers tooltip to future requirements).
- Active sort state is communicated visually via the chevron direction change. Screen reader
  announcement of sort state is not in scope for Phase 50.
