**1. Add cookie-based auth dependencies to app/auth/dependencies.py:**

Add two new functions (keep all existing functions unchanged):

- `get_current_user_cookie(access_token: str | None = Cookie(default=None)) -> User`:
  - If access_token is None, raise HTTPException 401
  - Decode JWT via decode_access_token(access_token)
  - Extract username from "sub" claim
  - Look up User by username, check enabled
  - Raise 401 on any failure
  - Import Cookie from fastapi at top of file

- `require_admin_cookie(user: User = Depends(get_current_user_cookie)) -> User`:
  - If user.role != "admin", raise 403
  - Return user

**2. Create app/admin/ui_router.py:**

Import Jinja2Templates, instantiate at module level: `templates = Jinja2Templates(directory="templates")`

Create `ui_router = APIRouter(prefix="/admin/ui", tags=["admin-ui"])`

Routes:

GET /admin/ui/login:
- Render templates/admin/login.html with empty error context
- No auth required

POST /admin/ui/login:
- Accept Form fields: username (str), password (str)
- Look up User, verify password, check enabled and role=="admin"
- On failure: re-render login.html with error="Invalid credentials or insufficient permissions"
- On success: create JWT via create_access_token, set HttpOnly cookie (key="access_token", httponly=True, samesite="lax"), redirect to /admin/ui/users with 302

GET /admin/ui/users (protected by require_admin_cookie):
- Fetch all users via User.find_all().to_list()
- Detect HX-Request header (hx_request: Annotated[str | None, Header()] = None)
- If hx_request: return users_table.html partial
- Else: return users.html full page
- Pass users list in context

POST /admin/ui/users/create (protected by require_admin_cookie):
- Accept Form fields: username, callsign, password
- Check duplicate username — if exists, return users_table.html with an error message via HX-Retarget or include error in context
- Create User with role="operator", enabled=True, callsign uppercased, hashed password
- Fetch updated user list, return users_table.html partial (HTMX will swap the table body)

POST /admin/ui/users/{username}/toggle (protected by require_admin_cookie):
- Look up user by username
- Toggle: new_enabled = not user.enabled
- LAST-ADMIN GUARD: If disabling and user.role == "admin", count enabled admins. If count <= 1, return an error fragment or set HX-Reswap header with error message.
- Use `await user.set({User.enabled: new_enabled})`
- Fetch updated user list, return users_table.html partial

POST /admin/ui/users/{username}/reset-password (protected by require_admin_cookie):
- Accept Form field: password
- Look up user — 404 if not found
- Use `await user.set({User.hashed_password: hash_password(password)})`
- Return users_table.html partial with success indication

GET /admin/ui/logout:
- Delete the access_token cookie
- Redirect to /admin/ui/login

CRITICAL: UI auth failures must redirect to login page, NOT return JSON. Wrap require_admin_cookie usage so that 401/403 from the dependency results in a redirect. Do this by catching HTTPException in a try/except within the route, or by using a custom exception handler for the /admin/ui prefix. Simplest approach: create a helper dependency `require_admin_cookie_or_redirect` that catches the HTTPException and raises a RedirectResponse. Alternatively, use `app.exception_handler` scoped to check the request path.

Recommended approach for auth-redirect: Use a middleware or a wrapper dependency:
```python
from fastapi.responses import RedirectResponse

async def require_admin_cookie_or_redirect(request: Request):
    try:
        return await require_admin_cookie(
            await get_current_user_cookie(request.cookies.get("access_token"))
        )
    except HTTPException:
        raise HTTPException(status_code=302, headers={"Location": "/admin/ui/login"})
```
Actually, FastAPI does not support redirect via HTTPException cleanly. Instead, use a simpler pattern: make each protected route call the cookie dependency manually and handle the redirect:

Better approach — just let the dependency raise 401/403 and add an exception handler on the app that checks if the request path starts with /admin/ui/ and returns RedirectResponse:
```python
# In app/main.py or ui_router.py
@app.exception_handler(HTTPException)
async def ui_auth_redirect(request: Request, exc: HTTPException):
    if request.url.path.startswith("/admin/ui/") and exc.status_code in (401, 403):
        return RedirectResponse(url="/admin/ui/login", status_code=302)
    return JSONResponse(status_code=exc.status_code, content={"detail": exc.detail})
```

Use this exception handler approach — add it in app/main.py after app creation.
