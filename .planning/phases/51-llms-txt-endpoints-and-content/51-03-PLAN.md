Endpoint inventory (authoritative — from RESEARCH.md lines 136-187):

1.  POST   /auth/token         — None (login) — OAuth2 form login; returns JWT access_token
2.  GET    /auth/me             — Bearer JWT — Returns authenticated user's username, callsign, role
3.  POST   /api/qsos/           — Bearer JWT or X-API-Key — Create QSO; 201 success, 409 duplicate
4.  GET    /api/qsos/           — Bearer JWT or X-API-Key — List QSOs (pagination + filters: call, band, mode, date, sort)
5.  GET    /api/qsos/{qso_id}   — Bearer JWT or X-API-Key — Fetch single QSO; 404 if not owned or deleted
6.  PATCH  /api/qsos/{qso_id}   — Bearer JWT or X-API-Key — Partial update; 404 if not owned
7.  DELETE /api/qsos/{qso_id}   — Bearer JWT or X-API-Key — Soft-delete; 204 on success
8.  POST   /api/adif/import     — Bearer JWT — Import .adi/.adif (multipart); returns import report JSON
9.  GET    /api/adif/export     — Bearer JWT — Stream full logbook as .adi file download
10. GET    /api/profile/        — Bearer JWT — Return operator's profile fields
11. PATCH  /api/profile/        — Bearer JWT — Update operator profile (partial)
12. POST   /api/tokens/         — Bearer JWT — Create named API token; returns full token once
13. GET    /api/tokens/         — Bearer JWT — List all active tokens for the operator
14. DELETE /api/tokens/{token_id} — Bearer JWT — Revoke token; 204 on success
15. GET    /health              — None — MongoDB health check; 200 ok or 503 error
16. GET    /api/whoami          — Bearer JWT — Returns operator callsign from JWT
