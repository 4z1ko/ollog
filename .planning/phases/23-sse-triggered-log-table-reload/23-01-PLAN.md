**Change C — Add script block for SSE event handling.**

Insert the following script block AFTER the `#log-table` div closing tag (after line 84) and BEFORE `{% endblock %}` (line 85):

```html
<script>
  (function() {
    var indicator = document.getElementById('live-indicator');

    // LIVE-05: Show LIVE indicator when SSE connection opens
    document.body.addEventListener('htmx:sseOpen', function(e) {
      if (e.detail.elt && e.detail.elt.id === 'log-table') {
        indicator.style.background = '#1e8449';
        indicator.textContent = 'LIVE';
        indicator.style.display = '';
      }
    });

    // LIVE-05: Show OFFLINE indicator on SSE error
    document.body.addEventListener('htmx:sseError', function(e) {
      if (e.detail.elt && e.detail.elt.id === 'log-table') {
        indicator.style.background = '#c0392b';
        indicator.textContent = 'OFFLINE';
        indicator.style.display = '';
      }
    });

    // LIVE-05: Hide indicator when SSE connection closes
    document.body.addEventListener('htmx:sseClose', function(e) {
      if (e.detail.elt && e.detail.elt.id === 'log-table') {
        indicator.style.display = 'none';
      }
    });

    // LIVE-01/02/03/04: Conditional refresh on new_qso SSE event
    document.body.addEventListener('htmx:sseMessage', function(e) {
      // Only respond to new_qso events on the log-table element
      if (!e.detail || !e.detail.elt || e.detail.elt.id !== 'log-table') return;
      if (e.detail.type !== 'new_qso') return;

      // LIVE-03: Server-side guard — only refresh on page 1, default sort, no filters
      if (!document.getElementById('auto-refresh-ok')) return;

      // LIVE-04: Client-side guard — do not refresh while edit row is open
      if (document.querySelector('#log-table input')) return;

      // LIVE-01/02: Trigger re-fetch of /log/view (JWT cookie sent automatically = operator isolation)
      htmx.ajax('GET', '/log/view', {target: '#log-table', swap: 'innerHTML'});
    });
  })();
</script>
```

**Important implementation notes:**
- The `htmx.ajax('GET', '/log/view', ...)` call uses the browser's cookie jar, which includes the JWT httpOnly cookie. This ensures operator isolation (LIVE-02) without any extra work.
- The `htmx:sseMessage` event listener approach is used instead of `hx-trigger="sse:new_qso [condition]"` because JS filter evaluation in hx-trigger SSE events has only medium confidence from research. The listener approach is fully reliable.
- Event detail property paths: use `e.detail.elt` (not `e.target`) for htmx SSE events. Use `e.detail.type` for the SSE event name.
- The `#auto-refresh-ok` span is inside `#log-table` innerHTML, so it gets replaced on every HTMX swap (pagination, filter, sort). This is correct — after any navigation that changes state, the server re-evaluates whether the marker should be present.
